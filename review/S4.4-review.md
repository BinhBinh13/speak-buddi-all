## Code Review

### BE — quiz_repo.py: `get_attempt_result()`
- **LATERAL subquery + LIMIT 1** đúng cách xử lý grammar_mapping (nhiều `is_correct=TRUE` → chỉ lấy 1 theo `display_order ASC`).
- `LEFT JOIN LATERAL ... ON TRUE` đảm bảo flashcard (không có `quiz_answer` row) vẫn trả về row với `correct_answer_text = NULL`.
- Pattern nhất quán với các hàm repo khác (raw SQL + text()).

### BE — schemas/quiz.py: `QuizAnswerReview` + `QuizAttemptResultOut`
- Fields đúng, Optional đúng, `Field(default_factory=list)` cho `answers`.
- `duration_seconds: Optional[int]` đủ cho UI.

### BE — routers/quiz.py: `GET /api/attempts/{attempt_id}/result`
- Guard đầy đủ: 404 → 403 → 409 theo thứ tự đúng.
- `max(0, int(delta.total_seconds()))` guard clock-skew âm.
- Log `GET_RESULT` đầy đủ thông tin.
- Import `QuizAttemptResultOut` + `QuizAnswerReview` đúng vị trí.

### FE — quizService.js: `getAttemptResult()`
- Pattern nhất quán với các hàm khác.
- Được import đúng trong `QuizResultPage.jsx`.

### FE — QuizResultPage.jsx
- **Loading skeleton**: có, bám palette đúng.
- **Error + retry**: có, nút retry đúng.
- **ScoreCircle**: `conic-gradient` formula đúng (`scorePercent/100 * 360` degrees).
- **Motivational message**: 4 ngưỡng (≥90/≥70/≥50/<50%) đúng plan.
- **StatsRow**: 3 cột, màu đúng DESIGN.md.
- **ReviewCard**: xử lý đúng 3 case: flashcard (tự đánh giá), đúng (hiện answer xanh), sai (gạch đỏ → xanh).
- **`formatDuration()`**: hiển thị phút/giây.
- **Animation `popIn`**: keyframe CSS inline, nhất quán với style file.
- `useParams()` lấy đủ `testId` + `attemptId`.

- `grammar_mapping` sai: `correct_answer_text` hiển thị 1 cặp đầu tiên (LIMIT 1) thay vì tất cả. Acceptable MVP; có thể mở rộng ở S4.5.
