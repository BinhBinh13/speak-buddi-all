## 3. Issues

### Issue 1 — Minor / Non-blocking: `get_test_by_id()` thiếu `level_code` trong JOIN

**File:** `speak-buddi-be/repositories/quiz_repo.py:119–137`

`get_test_by_id()` không JOIN bảng `level`, nên khi `GET /api/tests/{test_id}` trả `VocabularyTestOut`, field `level_code` luôn là `None`. Điều này không ảnh hưởng S4.5 vì `QuizPage.jsx` không dùng `level_code`, chỉ ảnh hưởng nếu tương lai có component cần hiển thị level trên màn làm bài.

**Mức độ:** Minor (không block AC nào, field `Optional` nên không gây lỗi runtime)

**Cách sửa (nếu cần):** Thêm `LEFT JOIN level l ON l.id = vt.level_id` và `l.code AS level_code` vào query trong `get_test_by_id()`.

### Issue 2 — Minor / Non-blocking: `fetchTests` race condition khi user thay đổi filter nhanh

**File:** `speak-buddi/src/features/quiz/QuizListPage.jsx:214–228`

`fetchTests` được khai báo với `useCallback` nhưng không có cancellation flag. Nếu user thay đổi filter nhanh, 2 requests có thể về theo thứ tự khác nhau. Tuy nhiên tác động thực tế thấp vì API response nhanh và context là educational app không có high-frequency interaction.

**Mức độ:** Minor (không block AC, MVP acceptable)

**Cách sửa (nếu muốn hardened):** Thêm `AbortController` trong `fetchTests` hoặc dùng debounce.

### Issue 3 — Observation: `topics` không load khi mount nếu `levels` rỗng

**File:** `speak-buddi/src/features/quiz/QuizListPage.jsx:206–209`

```js
if (levels.length > 0) {
  fetchTopics();
}
```

Nếu API `getLevels()` trả mảng rỗng, `topics` sẽ không bao giờ được load. Trong thực tế DB luôn có levels (A1–C2 seed sẵn), nên đây là edge case không ảnh hưởng flow thông thường.

**Mức độ:** Observation / không block
