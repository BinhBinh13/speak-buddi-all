- 3 endpoint đúng path/method/response_model: `GET /api/levels` → `list[LevelOut]`; `GET /api/topics` → `list[TopicListItemOut]`; `GET /api/topics/{topic_id}/words` → `list[TopicWordOut]`. ✅
- Topics query dùng **2 nhánh tách biệt** (có `level` / không `level`) — **không dùng `:level IS NULL`**, tránh lỗi asyncpg bind không xác định type. ✅
- 404 khi topic không tồn tại/inactive: query check `SELECT id FROM topic WHERE id=:tid AND is_active=TRUE`, `if check.first() is None → 404`. ✅
- Tags parse từ `json_agg`: xử lý cả trường hợp string (asyncpg) lẫn list, map sang `TagOut`. ✅
- `Depends(current_user)` gắn đủ cả 3 endpoint (`_user: dict = Depends(current_user)`). ✅
- `auth/deps.py` có đủ `_b64url`, `_verify`, `current_user`; `_verify` dùng `hmac.compare_digest`, kiểm tra `exp`; `current_user` chặn token type ≠ access. ✅
- `main.py`: `from routers import learning` + `app.include_router(learning.router)`. ✅
- Không log secret/token; chỉ `log.info("WORDS topic=... count=...")`. ✅
