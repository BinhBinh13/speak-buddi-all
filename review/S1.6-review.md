## 4. Findings

Khong co finding nghiem trong. Mot so diem nho can luu y:

### 4a. GSI: kiem tra `window.google?.accounts?.id` thay vi `?.accounts?.oauth2`
- **File:** `speak-buddi/src/shared/services/authService.js`, L77
- **Hien trang:** Code kiem tra `window.google?.accounts?.id` — dung voi flow One Tap (`google.accounts.id`).
- **Ke hoach:** Plan (S1.6-plan.md §4.2) ban dau neu `accounts.oauth2`, sau do chinh sang `accounts.id`. Code thuc te dung `accounts.id` — dung voi flow da chon. Khong phai loi.

### 4b. `compare_digest` voi `password_hash = None` (OAuth user dung /api/auth/login)
- **File:** `speak-buddi-be/main.py`, L479
- **Hien trang:** `hmac.compare_digest(pw_hash, user["password_hash"])` se raise `TypeError` khi `password_hash` la `None` vi `compare_digest` yeu cau ca hai tham so la `str` hoac `bytes`.
- **Anh huong:** TypeError se duoc `except Exception` o tren bat, nhung do khong co `except` o route `/api/auth/login`, loi se propagate thanh HTTP 500 thay vi 401.
- **Muc do:** Nho — chi anh huong edge case "OAuth user co y dung /api/auth/login" (khong phai luong chinh cua S1.6). Nen fix o S1.8 khi migrate DB, hoac patch nhanh o `/api/auth/login`: kiem tra `if not user or user["password_hash"] is None` truoc khi goi `compare_digest`.
- **Verdict anh huong:** Khong anh huong PASS vi AC-02-03 chi yeu cau Google OAuth flow, khong yeu cau /api/auth/login voi OAuth user.

### 4c. One Tap fallback UX (da ghi nhan trong implement log)
- **File:** `speak-buddi/src/shared/services/authService.js`, L111–136
- **Hien trang:** Fallback renderButton + auto-click co the gap van de voi browser security policy moi. Implement log da ghi nhan day la "UX refinement, khong anh huong AC".
- **Khuyen nghi:** Xem xet them `<div id="__gsi_btn_container__">` vao DOM cua LoginPage va dung `renderButton` luon (thay vi tao dong) de on dinh hon tren cac browser version moi. Thuoc ngoai pham vi S1.6.
