## Pattern đã bắt chước

- **Repository pattern:** `session_repo.py` — raw SQL `text()` + `db.execute` async, `mappings().all()`, log `log.info(...)` chỉ metadata an toàn.
- **Router pattern:** `session.py` — `APIRouter(prefix=...)`, `Depends(current_user)`, `Depends(get_db)`, FastAPI `Query()` validators.
- **Schema pattern:** `schemas/pronunciation.py` hiện có (Pydantic `BaseModel`, `Field(ge=..., le=...)`).
- **FE service pattern:** `pronunciationService.js` hiện có — `apiClient(...)` với `URLSearchParams` cho query params.

## Điểm tích hợp cho S6.2

S6.2 đã gọi `pronunciation_repo.insert_attempt(...)` trong `routers/pronunciation.py` POST `/score`. S6.3 bổ sung `GET /history` sử dụng `list_attempts(...)` mới — không thay đổi contract của S6.2.

Contract `GET /api/pronunciation/history`:
- Auth: Bearer JWT bắt buộc → 401 nếu thiếu
- Query: `?limit=20&offset=0` (limit 1–100, offset ≥ 0)
- Response: `{ items: [...], total, limit, offset }`
- Mỗi item: `{ id, target_text, overall_score, accuracy_score, fluency_score, feedback, created_at }`

## Cách test thủ công

1. Khởi động BE: `cd speak-buddi-be && uvicorn main:app --reload --port 8000`
2. Đăng nhập → lấy JWT token.
3. POST `/api/pronunciation/score` với body `{"target_text":"hello","transcript":"helo","topic_word_id":null}` → lấy `attempt_id` trả về.
4. GET `/api/pronunciation/history` với header `Authorization: Bearer <token>` → xác nhận trả list chứa attempt vừa tạo.
5. GET `/api/pronunciation/history?limit=5&offset=0` → kiểm tra phân trang.
6. GET `/api/pronunciation/history` không có token → 401.
7. Kiểm tra log BE: không thấy transcript/token trong log output.
