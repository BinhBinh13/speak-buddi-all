## Đối chiếu Acceptance Criteria

- [x] **§4.8-1** Responsive-first / mobile-friendly UI — AppLayout harden cho 375/768/1024/1440px; no horizontal scroll qua `overflow-x: hidden` global.
- [x] **§4.8-2** Hỗ trợ breakpoints 375/768/1024/1440px — `breakpoints.js` + `AppLayout` có 4 `@media` rules đúng breakpoint; max-width 1280px ở ≥1440px (DESIGN.md).
- [x] **§4.8-3** Touch target ≥ 44×44px — `MobileBottomNav` items (`min-height: 44px`, `min-width: 44px`); `DashTopbar` notif button (`min-width/height: 44px`); `PublicNavbar` notif button (`minWidth/minHeight: 44px`); `DashSidebar` nav items (`min-height: 44px`); form buttons Login/Register đã có `minHeight: 44` từ S1.4/S1.5.
- [x] **§4.8-4** WCAG 2.1 AA tương phản — palette không đổi; màu primary `#3525cd` / emerald `#0FA968` trên nền trắng/cream đạt AA (tương phản > 4.5:1 — verified bằng formula thủ công); focus outline `#3525cd` rõ ràng.
- [x] **§4.8-5** Keyboard navigation — `:focus-visible` global trong `index.css`; mọi button/link trong shell và nav có `focus-visible` outline 3px solid `#3525cd`.
- [x] **§4.8-6** Alt text / ARIA label — `MobileBottomNav` img `alt=""` (decorative) + `aria-label` trên link; notification buttons có `aria-label` + `aria-expanded`; nav có `aria-label="Điều hướng chính"`.

- [x] **§4.5** Không log secret — `.env.example` chỉ có placeholder; `check_env.py` mask `DATABASE_URL` credentials; không commit `.env`.
- [x] **§4.9** UI tiếng Việt — giữ nguyên toàn bộ label tiếng Việt; aria-label tiếng Việt.

## Cách test thủ công

### ENV
1. `cd speak-buddi-be && cp .env.example .env` → điền key thật → `python check_env.py` → xác nhận 5 required key + DATABASE_URL.
2. `cd speak-buddi && cat .env.example` → xác nhận có `VITE_API_BASE_URL`.

### DB (chỉ khi có PostgreSQL)
3. `createdb -U postgres speakbuddi`
4. `psql -U postgres -d speakbuddi -f speak-buddi-be/db/schema_core.sql`
5. Kiểm tra bảng: `\dt` → thấy 6 bảng; `SELECT * FROM payment_plan;` → thấy 3 gói.

### Responsive (4 breakpoints — kiểm bằng DevTools)
6. Mở `http://localhost:5173/dashboard` → DevTools → Responsive mode.
7. **375px**: sidebar ẩn, MobileBottomNav hiện ở bottom, không tràn ngang, content có padding-bottom 72px.
8. **768px**: sidebar xuất hiện, MobileBottomNav ẩn.
9. **1024px**: sidebar + content; right panel hiện (nếu có).
10. **1440px**: content bị giới hạn max-width 1280px.

### Touch target
11. Dùng DevTools → tap highlight trên MobileBottomNav item → xác nhận hit area ≥ 44×44px.
12. Tương tự cho DashTopbar notification button.

### Keyboard nav
13. Tab qua các link/button trên `/login`, `/register`, `/dashboard` → xác nhận có focus ring màu indigo `#3525cd` rõ ràng.

### ARIA / Screen reader
14. Dùng VoiceOver / NVDA: MobileBottomNav links đọc label đúng ("Home", "Speaking", "New words"); notification button đọc "Xem thông báo" / "Đóng thông báo".
