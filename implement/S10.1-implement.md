## Tóm tắt

Triển khai CRUD Admin cho `payment_plan`: 4 endpoint BE (`/api/admin/payment-plans`) + trang FE `/admin/payments` với grid card, modal create/edit, validation inline FE+BE theo §5.2. Tái dùng `AdminLayout`/`AdminSidebar` từ S9.1. Soft delete UI để dành S10.2.

## Quyết định implement

- **Giá = 0:** Cho phép gói miễn phí qua checkbox "Gói miễn phí"; gói trả phí (không tick) bắt `price_vnd > 0` ở FE.
- **Lỗi BE:** Trả `400` + `detail` là chuỗi message (prefix `⚠`); FE map qua `mapPaymentPlanApiErrorToFields()`.
- **Schema admin tách riêng:** `payment_plan_admin.py` — không sửa `schemas/payment.py` (pricing/checkout).

## Hướng verify cho Reviewer

1. Đăng nhập `admin@speakbuddi.com` / `Admin@123` → `/admin/payments`.
2. Tạo gói: name, price, duration, ≥1 feature → toast thành công, card mới.
3. Sửa giá → card cập nhật.
4. Submit thiếu name / 0 feature / giá âm → inline error, không lưu.
5. Gói trả phí với giá 0 (không tick miễn phí) → inline `⚠ Giá gói trả phí phải lớn hơn 0.`
6. Student token → `GET /api/admin/payment-plans` → 403.
7. So sánh UI với `quan_li_goi_thanh_toan_admin/screen.png`.
