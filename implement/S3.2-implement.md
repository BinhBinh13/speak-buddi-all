## Cách test thủ công

### Backend (yêu cầu DB đã chạy schema_learning.sql + seed)

```bash
# 1. Khởi động server
cd speak-buddi-be && uvicorn main:app --reload --port 8000

# 2. Lấy token
TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@speakbuddi.com","password":"password123"}' \
  | python -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

# 3. GET /api/levels
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8000/api/levels | python -m json.tool
# → 6 level A1–C2, sorted by display_order

# 4. GET /api/topics?level=A1
curl -s -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8000/api/topics?level=A1" | python -m json.tool
# → topic "Greetings & Introductions" với word_count=5

# 5. GET /api/topics/{id}/words (thay {id} bằng UUID từ bước 4)
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8000/api/topics/<TOPIC_ID>/words | python -m json.tool
# → 5 từ (hello, goodbye, name, meet, introduce) với tags

# 6. Test 404
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8000/api/topics/00000000-0000-0000-0000-000000000000/words
# → {"detail":"Topic not found or inactive"}
```

### Frontend

1. Chạy BE (`uvicorn main:app --reload`) + FE (`npm run dev`)
2. Đăng nhập với `demo@speakbuddi.com / password123`
3. Vào `http://localhost:5173/vocabulary`
4. Chọn level A1 từ dropdown → thấy card "Greetings & Introductions" với "5 từ"
5. Click card → chuyển sang flashcard mode, thấy progress bar "1/5 từ"
6. Thẻ "hello": thấy IPA `/həˈloʊ/`, nghĩa VI "xin chào", tag badge "everyday", nút audio
7. Click "Hiện nghĩa tiếng Anh" → hiện "a greeting used when meeting someone"
8. Click "Tiếp theo" / "Cần luyện thêm" / "Đã thuộc" → progress bar tăng
9. Từ không có `grammar_note` → block ghi chú KHÔNG render
10. Từ không có `meaning_en` → nút toggle KHÔNG hiện
11. Click "← Quay lại" → về topic selector
12. Chọn level B2 (không có topic seed) → hiện empty state
13. Không đăng nhập → truy cập `/vocabulary` → redirect `/login`
