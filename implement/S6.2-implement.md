## Cách test thủ công

1. **Chạy migration:** `psql -U <user> -d speakbuddi -f speak-buddi-be/db/migrations/007_pronunciation.sql`
2. **Khởi động backend:** `cd speak-buddi-be && uvicorn main:app --reload --port 8000`
3. **Khởi động frontend:** `cd speak-buddi && npm run dev`
4. **Đăng nhập** với tài khoản hợp lệ (có JWT).
5. Vào trang `/pronunciation`.
6. **Happy path:** Nhấn nút mic → đọc "Departure" → nhấn Stop → đợi loading → Score Display xuất hiện với gauge, progress bar Chính xác/Lưu loát, chip âm tiết (De/par/ture), feedback tiếng Việt.
7. **Nút "Thử lại":** nhấn → reset về idle.
8. **Nút "Từ tiếp theo":** nhấn → chuyển sang "Adventure", ghi âm lại được.
9. **Lỗi transcript rỗng:** tắt mic trước khi nói → banner `"🎤 Không nghe thấy gì..."`.
10. **Lỗi AI (test BE):** gọi `POST /api/pronunciation/score` với transcript rỗng `""` → expect 400; dùng key sai → expect 502.
11. **Không có JWT:** `curl POST /api/pronunciation/score` không có header → 401.
12. **Kiểm DB:** sau 1 attempt thành công, query:
    ```sql
    SELECT * FROM pronunciation_attempt ORDER BY created_at DESC LIMIT 1;
    SELECT * FROM pronunciation_syllable_score WHERE attempt_id = '<id>';
    ```
13. **Responsive:** resize 375px → bento chuyển 1 cột, nút ≥ 44px vẫn dễ nhấn.
14. **So sánh với screen.png:** gauge tổng màu xanh, chip xanh/cam đúng ngưỡng, layout bento giống mockup.
