## Đối chiếu Acceptance Criteria

- [x] **AC-03-04** — BE `PATCH /api/profile/level` persist `target_level` mới; FE gọi API → `updateUser({ level })` cập nhật AuthContext + localStorage; roadmap (computed on-the-fly từ `target_level`) tự phản ánh level mới khi gọi tiếp theo.
- [x] **BR09 (BE)** — validate `level.strip().upper() not in {"A1","A2","B1","B2","C1","C2"}` → HTTP 400 + message tiếng Việt; chuẩn hóa lowercase → uppercase trước khi validate nên `"b1"` → 200.
- [x] **BR09 (FE)** — `LevelSelector` chỉ render 6 giá trị hợp lệ; không thể submit giá trị ngoài tập.
- [x] **interests không bị reset** — `update_level()` dùng câu UPDATE chỉ đụng `target_level + updated_at`, không chạm `interests/daily_minutes/words_per_session`.
- [x] **Confirm flow** — nút "Lưu thay đổi" disabled khi `selectedLevel === currentLevel`; click → `Modal` confirm trước khi gọi API.
- [x] **Auth guard** — endpoint `Depends(current_user)` → 401 nếu không có JWT hợp lệ.
- [x] **Log an toàn** — chỉ log `user_id` + `level`, không log interests/token/email (SRS §4.5).
- [x] **Mockup bám sát** — layout 2 cột (sticky nav + sections), card trắng với shadow nhẹ, badge CEFR pill, primary `#3525cd`, font Be Vietnam Pro.

- [x] **Touch target** — nút "Lưu thay đổi" `minHeight: 44px`; mỗi `LevelCard` `minHeight: 90px`.

## Cách test thủ công

1. **Khởi động:**
   ```bash
   cd speak-buddi-be && uvicorn main:app --reload --port 8000
   cd speak-buddi && npm run dev
   ```

2. **AC-03-04 (BE):**
   ```http
   PATCH /api/profile/level
   Authorization: Bearer <valid_access_token>
   {"level": "B1"}
   → 200 {"level":"B1","onboarding_completed":true}
   ```
   Truy DB: `SELECT target_level, interests FROM user_profile WHERE user_id = '...'`
   → `target_level = 'B1'`; `interests` **không thay đổi**.

3. **BR09 (BE):**
   ```http
   PATCH /api/profile/level {"level": "X9"} → 400 "⚠ Trình độ không hợp lệ..."
   PATCH /api/profile/level {"level": "b1"} → 200 {"level":"B1",...}
   ```

4. **Auth guard:**
   ```http
   PATCH /api/profile/level (không có token) → 401
   ```

5. **FE end-to-end:**
   - Đăng nhập → vào `/profile`
   - Thấy level hiện tại được highlight trong selector
   - Nút "Lưu thay đổi" disabled khi chọn cùng level cũ
   - Chọn level khác → nút enable → click → Modal confirm hiện ra
   - Xác nhận → toast "Đã cập nhật trình độ thành công!"
   - Kiểm tra level hiện tại trong LevelDisplay cập nhật ngay
   - Kiểm tra roadmap/dashboard → nội dung theo level mới (on-the-fly)

6. **Responsive:** kiểm tra tại 375px, 768px, 1440px — level cards wrap thành 1 hoặc 2 cột tùy viewport.
