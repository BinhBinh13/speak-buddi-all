## Cách test thủ công

1. Chạy migration: `psql -U postgres -d speakbuddi -f speak-buddi-be/db/migrations/006_user_session_progress.sql`
2. Khởi động BE: `cd speak-buddi-be && uvicorn main:app --reload --port 8000`
3. Khởi động FE: `cd speak-buddi && npm run dev`
4. Đăng nhập → vào `/roadmap`
5. Click 1 node topic không bị locked → modal mở đúng (tên, level badge, số từ)
6. Esc / click backdrop / nút X → modal đóng
7. Bấm "+ Thêm vào từ vựng" → UI đổi thành "✓ Đã thêm" ngay; bấm lại → đảo về "Thêm"
8. F5 lại → trạng thái add/remove vẫn đúng (đồng bộ DB)
9. Bấm "Học từ vựng" → navigate `/learn/<slug>`; VocabularyPage hiện flashcard topic đó (không qua dropdown)
10. Bấm "Luyện với AI" → URL `/conversation`; kiểm tra `location.state` trong ConversationPlaceholder có `topicId, topicName, batchIndex, words[]` với `words.length ≤ words_per_session`
11. Vào `/vocabulary` khi chưa add topic nào → hiện empty state + link "Vào Lộ trình"
12. Sau khi add topic → `/vocabulary` hiện card đúng topic đó

```bash
# Add topic
curl -X POST http://localhost:8000/api/user/topics \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"topic_id": "<uuid>"}'

# List user topics
curl http://localhost:8000/api/user/topics \
  -H "Authorization: Bearer <token>"

# Session summary
curl http://localhost:8000/api/topics/<uuid>/session-summary \
  -H "Authorization: Bearer <token>"

# Upsert session progress
curl -X PUT http://localhost:8000/api/topics/<uuid>/sessions/progress \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"batch_index": 0, "batch_size": 5, "status": "in_progress"}'

# Delete user topic → 204
curl -X DELETE http://localhost:8000/api/user/topics/<uuid> \
  -H "Authorization: Bearer <token>"
```
