## Cách implement snake layout

Theo mockup `lo_trinh_hoc_tap_snake_style`:
- Container flex-column + alignItems center + overflowX hidden
- Mỗi "row" là div relative, kích thước 320×160px (thu nhỏ ở tablet)
- Node tròn 96×96px đặt `position:absolute top:0 left:0` (index chẵn) hoặc `right:0` (index lẻ)
- Connector SVG Bezier (viewBox 0 0 100 100, preserveAspectRatio=none):
  - Left→Right: `M 0 0 C 60 0, 40 100, 100 100` (bám mockup chính xác)
  - Right→Left: `M 100 0 C 40 0, 60 100, 0 100`
- Màu connector: `#a7f3d0` nếu node nguồn = completed; `#e4e1ee` cho các trạng thái khác
- Mobile ≤480px: đơn giản hóa → node canh center, bỏ zigzag (danh sách thẳng)
- Palette màu node `available` (vì BE S2.2 trả placeholder): 6 màu xoay vòng (indigo → sky → violet → cyan → primary → purple) giúp roadmap trông đẹp khi tất cả available

## Cách test thủ công

1. **Loading state:** `cd speak-buddi && npm run dev` → vào `/roadmap` với Network throttle "Slow 3G" → thấy skeleton pulse header + 4 node placeholder
2. **Data state:** Đăng nhập user đã onboarding (level có topic) → `/roadmap` → thấy header "Lộ trình của bạn — {level_name}", các node xếp zigzag, connector cong giữa node
3. **Empty state:** Dùng user có level không có topic active hoặc mock `nodes: []` từ BE → thấy `RoadmapHeader` vẫn hiển thị + `RoadmapEmptyState` bên dưới với nút "Cập nhật trình độ" → /profile
4. **Error state:** Tắt BE (hoặc trả 500) → thấy error box đỏ + nút "Thử lại" → click retry → gọi lại API
5. **Auth guard:** Truy cập `/roadmap` chưa đăng nhập → ProtectedRoute đẩy về `/login`; xóa token → apiClient redirect login
6. **Sidebar active:** Khi ở `/roadmap` → item "Roadmap" trong DashSidebar có active state; MobileBottomNav hiện item Roadmap ở mobile
7. **Sau onboarding:** Hoàn thành bước 4 onboarding → redirect tới `/roadmap` (không phải /dashboard)
8. **Responsive:** Resize 375/768/1024/1440 → snake không tràn ngang, node ≥ 44px touch target, MobileBottomNav ẩn/hiện đúng
