## Đối chiếu Acceptance Criteria

- [x] AC-1 — `POST /attempts/{id}/submit` trả `score_percent` đúng công thức: đã có từ S4.3, `calculate_score()` trong `quiz_service.py`
- [x] AC-2 — Attempt đã submit: `status='submitted'`, `submitted_at!=NULL`: `submit_attempt()` repo set cả 2 (S4.3)
- [x] AC-3 — Attempt sau submit không thể submit lại: guard `status != 'in_progress'` → 409 trong `submit_quiz` (S4.3)
- [x] AC-4 — `GET /api/attempts/{id}/result` trả đúng score + answers: endpoint mới trả `QuizAttemptResultOut` với `score_percent`, `correct_answers`, `wrong_answers`, `answers[]`
- [x] AC-5 — 403 khi user khác fetch attempt: kiểm tra `attempt["user_id"] != user["sub"]` → 403
- [x] AC-6 — 409 khi fetch result của attempt `in_progress`: kiểm tra `status != "submitted"` → 409
- [x] AC-7 — FE: màn kết quả hiển thị score circle với % đúng: `ScoreCircle` component dùng `conic-gradient(#006c49 {angle}deg, #eae6f4 0deg)` với `angle = scorePercent/100*360`
- [x] AC-8 — FE: stats row hiển thị đúng số đúng/sai/tổng: `StatsRow` render `correct_answers`, `wrong_answers`, `total_questions`
- [x] AC-9 — FE: review list hiển thị question_text + correct/incorrect marker: `ReviewCard` với border-left xanh/đỏ, icon FiCheckCircle/FiXCircle
- [x] AC-10 — FE: nút "Làm lại" navigate về `/quiz/:testId`: `navigate(\`/quiz/${testId}\`)`
- [x] AC-11 — BR08: row `quiz_attempt` không bị UPDATE sau submit: `get_attempt_result` chỉ SELECT, không có UPDATE sau `submit_attempt()`

## Cách test thủ công

1. Khởi động BE: `cd speak-buddi-be && uvicorn main:app --reload --port 8000`
2. Khởi động FE: `cd speak-buddi && npm run dev`
3. Đăng nhập → vào 1 bài kiểm tra có dữ liệu → làm bài → nộp → tự động redirect sang `/quiz/:testId/result/:attemptId`
4. Kiểm tra màn kết quả:
   - Score circle hiển thị % với màu conic-gradient xanh emerald
   - Stats row: số đúng (xanh), sai (đỏ), tổng (indigo)
   - Nút "Làm lại" → quay lại `/quiz/:testId`
   - Nút "Tiếp tục học" → navigate(-2)
   - Link "Về lộ trình" → navigate("/")
   - Review list: border-left xanh cho câu đúng, đỏ cho câu sai
5. Test API trực tiếp:
   - `GET /api/attempts/{id}/result` với token đúng user → 200 + JSON
   - Thử với attempt `in_progress` → 409
   - Thử với token user khác → 403
   - Thử với attempt không tồn tại → 404
