## Điều chỉnh so với plan gốc

Plan gốc đề xuất `resolve_db_user_id` (lazy-provision) vì lo ngại JWT `sub = "u_001"` không phải UUID. Theo hướng dẫn từ prompt:

- `MOCK_USERS["demo@speakbuddi.com"]["id"]` đã là `"00000000-0000-0000-0000-000000000002"` (UUID hợp lệ).
- `seed_dev.sql` đã seed user này vào bảng `users`.
- **Không cần lazy-provision.** Code dùng thẳng `uuid.UUID(payload["sub"])` làm `user_id` FK.

### Test upsert API (psql + Network tab)

1. Khởi động BE: `cd speak-buddi-be && uvicorn main:app --reload --port 8000`
2. Đăng nhập FE với `demo@speakbuddi.com / password123` → lấy token.
3. Mở tab học từ vựng, chọn topic, nhấn "Đã thuộc" một từ.
4. Network tab → PUT `/api/topics/{id}/words/{id}/progress` → status 200, body có `status: "known"`, `review_count: 1`.
5. Nhấn lại "Đã thuộc" cùng từ đó → `review_count: 2` (upsert tăng).
6. Kiểm tra DB:
   ```sql
   SELECT * FROM user_word_progress
   WHERE user_id = '00000000-0000-0000-0000-000000000002';
   ```
   → thấy row có `status`, `topic_id`, `level_id` đúng.

### Test GET progress (hydrate trạng thái)

1. Thoát topic → vào lại cùng topic.
2. Network tab → GET `/api/topics/{id}/progress` → `known_count` đúng, `words[]` chứa từ đã đánh dấu.
3. Badge "Đã thuộc" hiển thị trên flashcard của từ đã lưu.
4. Progress bar hiển thị `"Đã thuộc: N/total (X%)"`.

### Test edge cases

- Gọi PUT với `word_id` ngẫu nhiên không thuộc topic → 404 `"Word not found in topic or inactive"`.
- Gọi GET progress với topic id không tồn tại → 404.
- Gọi không có token → 401.
