## Quyết định thiết kế

- **Fixed 5h window** (không sliding): `window_start_at` cố định lúc tạo, `window_end_at = window_start_at + 5h`. Reset sau khi cửa sổ hết hạn (lazy — không cần cron).
- **Lazy reset (BR03)**: `get_or_create_active_window` chỉ tìm `window_end_at > NOW()`. Cửa sổ cũ hết hạn tự bị bỏ qua; INSERT cửa sổ mới với `used_seconds=0`.
- **Đếm thời gian**: FE gửi `elapsed_seconds` trong `SpeakRequest`. Nếu FE không gửi (= 0) → backend fallback 30 giây/turn (`_FALLBACK_DELTA_SECONDS = 30`). Cơ chế heartbeat chính xác hơn là TBD (ghi ở mục Nợ kỹ thuật).
- **Hằng số tập trung**: `_DEFAULT_WINDOW_SECONDS = 18_000`, `_DEFAULT_MAX_SECONDS = 900` trong `quota_repo.py`; không hardcode rải rác.
- **`UNIQUE (user_id, window_start_at)`**: dùng `ON CONFLICT ... DO UPDATE` để tránh race condition khi 2 request đồng thời tạo cửa sổ mới.
- **QuotaBar màu cảnh báo**: chuyển từ `C.primary` (indigo) sang `C.warning` (amber `#f59e0b`) khi `remaining_seconds <= 300` (còn 5 phút).

## Cách test thủ công

**AC-09-01 (free user còn quota):**
1. Đăng nhập với tài khoản free (hoặc không set `is_paid=true` trong JWT).
2. Vào màn hội thoại — header hiển thị QuotaBar "⏱ Còn X phút".
3. Gửi tin nhắn → nhận phản hồi AI.
4. Gọi `GET /api/ai/quota` → kiểm tra `used_seconds` tăng lên (khoảng +30s/turn nếu FE không gửi `elapsed_seconds`).

**AC-09-02 (quota hết → 429):**
1. Dùng SQL để seed `used_seconds = 900` cho user:
   ```sql
   UPDATE ai_quota_window SET used_seconds = 900
   WHERE user_id = '<user_uuid>' AND window_end_at > NOW();
   ```
2. Gửi tin nhắn → backend trả 429, FE hiển thị QuotaBanner màu amber.
3. Kiểm tra: input textarea placeholder thay đổi thành "Quota đã hết...", textarea và nút gửi bị disabled.
4. Nút "Nâng cấp Pro" → navigate đến `/pricing`.
5. Kiểm tra thời điểm reset hiển thị đúng (localeTimeString vi-VN).

**BR03 (lazy reset sau 5h):**
1. Seed một window với `window_end_at = NOW() - interval '1 second'` (đã hết hạn) và `used_seconds = 900`.
2. Gọi `GET /api/ai/quota` → trả cửa sổ mới với `used_seconds = 0`, `remaining_seconds = 900`.
3. Gửi tin nhắn → thành công (không bị 429).

**BR05/AC-09-03 (paid user không bị quota):**
1. Đăng nhập với tài khoản có `is_paid=true` trong JWT.
2. Gọi `GET /api/ai/quota` → `{ is_paid: true, unlimited: true }`.
3. Vào màn hội thoại — không có QuotaBar trong header.
4. Gửi tin nhắn nhiều lần → không bao giờ nhận 429.

**DevTools test 429:**
1. Trong DevTools Network → chặn/mock request `POST /speak` → trả 429 với body `{ detail: { code: "quota_exceeded", message: "⏱ Bạn đã dùng hết 15 phút luyện nói. Vui lòng quay lại sau khi quota được reset hoặc nâng lên Pro để luyện không giới hạn.", reset_at: "<iso>" } }`.
2. Kiểm tra QuotaBanner xuất hiện với đúng message + thời gian reset.
