- **Epic:** EPIC-2 — Onboarding & Lộ trình học (UC03, UC04 · F03, F04 · BR09)
- **Ưu tiên:** P1

## 1. Mục tiêu & phạm vi

Xây màn hình **Roadmap** (frontend only) hiển thị lộ trình học cá nhân hóa theo level đã chọn ở onboarding, vẽ dưới dạng **snake/zigzag path** (node trái → phải xen kẽ) bám mockup `lo_trinh_hoc_tap_snake_style`. Dữ liệu lấy từ `GET /api/roadmap` (đã có ở S2.2). Phải xử lý đủ 3 trạng thái runtime: **loading**, **error**, và **empty state** (level chưa có topic active). Backend không đổi.

"Done" = mở `/roadmap` (route protected) thấy header level + danh sách topic xếp snake với badge trạng thái (completed/active/locked, đánh dấu topic user đã chọn), loading skeleton khi tải, empty state khi `nodes` rỗng, error state khi API lỗi, responsive 375/768/1024/1440. Sau onboarding và sau đổi level (S2.3) roadmap phản ánh đúng level mới (do BE compute on-the-fly).

- [ ] **AC-04-01:** User đã onboarding + chọn level A1–C2 → mở roadmap thấy lộ trình cá nhân hóa với các topic khớp level. (header hiển thị `level` + `level_name`; render `nodes` theo `order_index`)

## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)
- **Không thay đổi.** `GET /api/roadmap` đã có từ S2.2, trả `RoadmapOut` (xem data contract bên dưới). Nếu trong lúc làm phát hiện response thiếu field cần cho UI → ghi vào mục Rủi ro, không tự sửa BE trong story này.

### Data contract (từ `speak-buddi-be/schemas/roadmap.py` & `roadmapService.js`)
`RoadmapOut`:
- `level: string` (vd "B1"), `level_name: string` (vd "Intermediate (B1)")
- `total_topics: number`, `selected_topics: number`
- `nodes: RoadmapNode[]`

`RoadmapNode` (field FE dùng):
- `id` — key React
- `name` — nhãn dưới node
- `slug` — (dự phòng cho deep-link sau; chưa điều hướng ở story này)
- `description` — tooltip/aria-label (optional)
- `order_index` — **quyết định thứ tự vẽ snake** (0-based). FE sort theo field này phòng BE trả không theo thứ tự.
- `is_interest` — topic user đã chọn khi onboarding → highlight/đánh dấu riêng
- `status` — `available | completed | in_progress | locked` → map sang style node (màu, badge check/lock, ring active). Lưu ý: tracking thật chưa có (S2.2 trả placeholder "available") → UI phải hoạt động đúng kể cả khi tất cả là "available".
- `word_count` — phụ đề nhỏ "N từ" (optional)

### Logic snake layout (zigzag)
Bám mockup: mỗi "row" cao ~160px, node tròn ~96px, đường cong SVG nối node này sang node kế. Quy tắc:
- Sort `nodes` theo `order_index` tăng dần.
- Với mỗi node theo index `i`: `i % 2 === 0` → **canh trái**, `i % 2 === 1` → **canh phải** (`alignSelf`/`marginLeft auto`/`marginRight auto` hoặc flex `justify-content`).
- **Connector cong** giữa node `i` và `i+1`: SVG `<path>` Bezier (mockup dùng `M 0 0 C 60 0, 40 100, 100 100` cho left→right, mirror cho right→left). MVP có thể đơn giản hóa: dùng SVG cong như mockup, hoặc fallback đường nối thẳng/dotted nếu SVG phức tạp — ưu tiên giữ cảm giác zigzag.
- Màu connector: xanh (`#a7f3d0` secondary-fixed) nếu node nguồn `completed`; xám (`#e4e1ee`) nếu dẫn tới node `locked`/`available`.
- Container: `display:flex; flexDirection:column; alignItems:center; overflowX:hidden`, mỗi row width cố định ~320px trên desktop, co lại trên mobile (dùng `clamp`/`maxWidth:100%`).

### Mapping `status` → style node (bám mockup + DESIGN.md)
- `completed` → nền màu tươi (mockup dùng các màu node `#f87171/#06b6d4/#ec4899`), badge tròn góc trên-phải nền secondary `#006c49` + icon check trắng.
- `in_progress` / node hiện hành → nền cam `#f59e0b` + ring `box-shadow: 0 0 0 6px #fcf8ff, 0 0 0 10px rgba(245,158,11,0.3)`, không badge.
- `locked` → nền `surface-variant #e4e1ee`, border `outline-variant`, `opacity ~0.7` + grayscale, badge lock.
- `available` (mặc định/placeholder S2.2) → nền primary-container `#4f46e5` hoặc màu trung tính tươi, không badge khóa (vì chưa có tracking thật). Cần chọn 1 style "available" rõ ràng để khi MỌI node = available màn vẫn đẹp.
- `is_interest === true` → thêm dấu hiệu phụ (vd viền/sao nhỏ) để phân biệt "topic bạn đã chọn".
- **Icon mỗi node:** mockup dùng Material Symbols theo từng topic. FE dự án dùng `react-icons` → chọn 1 icon mặc định (vd `BsBookmarkStar`/`BsCircleFill`) hoặc map vài icon theo slug; không bắt buộc khớp 1-1 với mockup, miễn nhất quán.

### Loading / Error state
- **Loading:** `RoadmapSkeleton` (header giả + 3–4 node pulse) — pattern `@keyframes pulse` như `VocabularyPage`.
- **Error:** box `background:#ffdad6; color:#ba1a1a; borderRadius:12` + nút "Thử lại" gọi lại `getRoadmap()`. (401 đã được `apiClient` tự xử lý → redirect login, phục vụ AC-04-03.)

### Routing
- Dùng **`/roadmap`** (đã có placeholder comment trong `App.jsx`, sidebar mockup có item Roadmap riêng). `/dashboard` vẫn là trang riêng — KHÔNG gộp.
- Đặt trong `<ProtectedRoute>` (AC-04-03).
- **Redirect sau onboarding:** hiện `OnboardingPage.jsx:122` redirect `/dashboard`. Đề xuất đổi sang `/roadmap` để khớp yêu cầu story ("sau onboarding user được redirect về đây"). Xác nhận với team trước khi đổi (xem Rủi ro); nếu giữ `/dashboard` thì roadmap vẫn truy cập qua sidebar.
- **Sau đổi level (S2.3):** không cần state đồng bộ — roadmap fetch mới mỗi lần mount, BE compute theo level mới nên tự cập nhật (AC-04-02). Đảm bảo fetch trong `useEffect` mount, không cache cứng.
