## 1. Mục tiêu & phạm vi

Cho phép Student mở một topic thuộc level của mình và học từ vựng dưới dạng flashcard: hiển thị từ, phiên âm IPA, nghĩa tiếng Việt (luôn hiện), nghĩa tiếng Anh (toggle), câu ví dụ, ghi chú ngữ pháp, và phát audio. Story cung cấp **2 API đọc dữ liệu** (list topics theo level, list words theo topic) trên schema S3.1 đã có, cộng **trang `/vocabulary` ở FE** port từ mockup `hoc_tu_vung_desktop`.

**Định nghĩa "done":** Student chọn level → thấy danh sách topic active của level đó → mở topic → lướt qua từng từ với đầy đủ thông tin học (nghĩa, ví dụ/grammar), phát audio, toggle nghĩa EN. Chỉ nội dung `is_active = TRUE` được hiển thị. Lưu tiến độ (mark "đã thuộc"/"cần luyện") **KHÔNG thuộc phạm vi S3.2** — đó là S3.3 (các nút vẫn render theo mockup nhưng chỉ điều hướng client-side giữa các từ, chưa gọi API lưu).

## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)

Backend hiện gom toàn bộ route trong `main.py`, chưa có thư mục `routers/`. Để giữ `main.py` gọn và mở đường cho Epic 3/4/9, **tạo APIRouter mới** và `include_router` vào app.

**File mới: `speak-buddi-be/routers/__init__.py`** (rỗng, đánh dấu package).

**File mới: `speak-buddi-be/routers/learning.py`** — `APIRouter(prefix="/api", tags=["learning"])` với 3 endpoint đọc:

1. `GET /api/levels` → `list[LevelOut]`
   - Trả 6 level A1–C2 sắp theo `display_order`. Dùng cho dropdown/filter chọn level ở FE.
   - SQL: `SELECT id, code, name, display_order FROM level ORDER BY display_order`.

2. `GET /api/topics?level=<code>` → `list[TopicOut]`
   - Query param `level` (code A1–C2, optional). Nếu có → lọc theo `level.code`. Chỉ trả topic `is_active = TRUE`, sắp theo `display_order`.
   - SQL (SQLAlchemy `text()` async):
     ```sql
     SELECT t.id, t.level_id, t.name, t.slug, t.description, t.display_order,
            t.source, t.is_active, t.created_at
     FROM topic t
     LEFT JOIN level l ON l.id = t.level_id
     WHERE t.is_active = TRUE
       AND (:level IS NULL OR l.code = :level)
     ORDER BY t.display_order, t.name;
     ```
   - Có thể thêm `word_count` (đếm topic_word active) để FE hiện "50 từ" như mockup — mở rộng `TopicOut` với field `word_count: int = 0` hoặc dùng schema riêng `TopicListItemOut`.

3. `GET /api/topics/{topic_id}/words` → `list[TopicWordOut]`
   - Trả các từ `is_active = TRUE` của topic, sắp theo `display_order`. Nếu topic không tồn tại / không active → `404`.
   - Mỗi từ kèm `tags` (join `topic_word_tag` → `tag`). Vì async raw SQL, gom tags bằng 1 query phụ theo danh sách `topic_word.id` rồi map vào, hoặc dùng `array_agg`/`json_agg` trong 1 query:
     ```sql
     SELECT tw.*, COALESCE(
       json_agg(json_build_object('id', tg.id, 'name', tg.name, 'slug', tg.slug))
         FILTER (WHERE tg.id IS NOT NULL), '[]') AS tags
     FROM topic_word tw
     LEFT JOIN topic_word_tag twt ON twt.topic_word_id = tw.id
     LEFT JOIN tag tg ON tg.id = twt.tag_id
     WHERE tw.topic_id = :topic_id AND tw.is_active = TRUE
     GROUP BY tw.id
     ORDER BY tw.display_order;
     ```

**Wiring trong `main.py`:**
- `from routers import learning` và `app.include_router(learning.router)`.
- Dùng `Depends(get_db)` (đã có ở `db/connection.py`) cho mỗi route → `AsyncSession`.
- Tái dùng schema có sẵn ở `schemas/learning.py` (`LevelOut`, `TopicOut`, `TopicWordOut`, `TagOut`) — đã khớp cột schema S3.1. Chỉ bổ sung `word_count` cho list topic nếu cần.

**Bảo vệ route (auth):** Theo SRS UC05, học từ vựng yêu cầu đăng nhập. Có thể thêm `Depends(current_user)` để khớp các route protected khác. Nếu muốn cho phép browse public ở MVP → ghi rõ TBD (mục 6) và thống nhất với user. Khuyến nghị: **gắn `current_user`** để nhất quán với `/api/auth/me`.

> Không tạo route ghi/sửa (CRUD admin) ở đây — đó là S9.1.

### Frontend (speak-buddi/)

Theo cấu trúc feature-based hiện có (`src/features/<feature>/`). Tạo feature mới `vocabulary`.

**File mới:**
