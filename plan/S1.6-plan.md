## 1. Tổng quan approach

### Approach được chọn: Option A — Frontend-driven (Google Identity Services / GSI)

**Lý do chọn:**

| Tiêu chí | Option A (GSI, FE-driven) | Option B (BE Authorization Code) |
|---|---|---|
| Phù hợp SPA React | **Tốt** — không cần redirect round-trip BE | Kém — phá vỡ SPA flow |
| Backend stateless (hiện tại) | **Tốt** — BE chỉ verify `id_token` | Kém — cần session/callback URL |
| Số lượng thay đổi BE | Nhỏ (1 endpoint mới) | Lớn (flow 3-leg, callback route) |
| Tương thích MOCK_USERS | **Có** — thiết kế dual-path rõ ràng | Khó hơn |
| Bảo mật | Đủ tốt cho v1 (HTTPS + token verify) | Cao hơn nhưng phức tạp hơn |

**Flow tóm tắt:**
1. Người dùng nhấn "Tiếp tục với Google" ở `LoginPage.jsx`.
2. Google Identity Services (GSI) hiện popup/redirect → người dùng chọn tài khoản Google.
3. GSI trả về `id_token` (JWT do Google ký) về cho FE.
4. FE gọi `POST /api/auth/google` với `{ id_token }`.
5. BE verify `id_token` bằng thư viện `google-auth` (check signature, aud, exp).
6. BE trích xuất `sub` (Google UID), `email`, `name`, `picture` từ payload.
7. BE kiểm tra `oauth_account` (MOCK hoặc DB):
   - Nếu `google / sub` chưa tồn tại + email chưa tồn tại → **tạo user mới + oauth_account**.
   - Nếu `google / sub` chưa tồn tại + email đã tồn tại → **link oauth_account vào user cũ**.
   - Nếu `google / sub` đã tồn tại → **login trực tiếp**.
8. BE phát JWT access + refresh token theo format hiện tại.
9. FE nhận token → lưu vào localStorage, navigate đến `/dashboard`.

### Tương thích MOCK_USERS / DB

Backend hiện dùng `MOCK_USERS` (dict in-RAM). S1.8 sẽ chuyển sang DB thật.

**Quyết định:** S1.6 dùng `OAUTH_ACCOUNTS` dict in-RAM (giống `MOCK_USERS`), code tách riêng thành helper, cấu trúc mirror `oauth_account` table (provider, provider_user_id, user_id). Khi S1.8 migrate sang DB thật, chỉ cần thay helper functions — không đụng route handler.

### 3.3 Logic tạo/link user (helper functions)

Tách thành helper riêng để dễ swap khi S1.8 dùng DB:

```python
# --- In-RAM store (song song với MOCK_USERS, xóa khi S1.8 done) ---
OAUTH_ACCOUNTS: dict[str, dict] = {
    # key: f"google:{provider_user_id}"
    # value: { "user_id": ..., "provider": "google", "provider_user_id": ... }
}

def _find_user_by_oauth(provider: str, provider_uid: str) -> dict | None:
    """Tra user qua oauth_account (MOCK — S1.8: query DB)."""
    record = OAUTH_ACCOUNTS.get(f"{provider}:{provider_uid}")
    if not record:
        return None
    # Tra ngược MOCK_USERS bằng id
    return next((u for u in MOCK_USERS.values() if u["id"] == record["user_id"]), None)

def _find_user_by_email(email: str) -> dict | None:
    """Tra user theo email (MOCK — S1.8: query DB)."""
    return MOCK_USERS.get(email)

def _create_user_from_google(google_sub: str, email: str, name: str) -> dict:
    """Tạo user mới từ Google OAuth (MOCK — S1.8: INSERT DB)."""
    new_id = f"u_{len(MOCK_USERS) + 1:03d}"
    MOCK_USERS[email] = {
        "id": new_id, "name": name, "email": email,
        "password_hash": None,   # NULL vì đăng ký qua OAuth
        "role": "student", "level": None, "streak": 0, "goal": None,
    }
    OAUTH_ACCOUNTS[f"google:{google_sub}"] = {
        "user_id": new_id, "provider": "google", "provider_user_id": google_sub,
    }
    return MOCK_USERS[email]

def _link_oauth_account(user: dict, provider: str, provider_uid: str) -> None:
    """Gắn oauth_account vào user đã tồn tại (MOCK — S1.8: INSERT oauth_account DB)."""
    OAUTH_ACCOUNTS[f"{provider}:{provider_uid}"] = {
        "user_id": user["id"], "provider": provider, "provider_user_id": provider_uid,
    }
```

**Route handler logic (`/api/auth/google`):**

```python
@app.post("/api/auth/google")
async def google_oauth(req: GoogleAuthRequest):
    # 1. Verify id_token
    google_payload = _verify_google_id_token(req.id_token)
    # → raises HTTPException 400/401/502 on failure

    google_sub = google_payload["sub"]
    email      = google_payload["email"].lower()
    name       = google_payload.get("name", email.split("@")[0])

    # 2. Lookup / create / link
    user = _find_user_by_oauth("google", google_sub)

    if user is None:
        existing = _find_user_by_email(email)
        if existing:
            _link_oauth_account(existing, "google", google_sub)
            user = existing
        else:
            user = _create_user_from_google(google_sub, email, name)

    # 3. Kiểm tra suspended
    if user.get("role") == "suspended" or user.get("status") == "suspended":
        raise HTTPException(status_code=403, detail="Tài khoản đã bị khóa.")

    # 4. Phát token
    access_token  = _make_access(user)
    refresh_token = _make_refresh(user)
    log.info("GOOGLE_AUTH ok user=%s", user["id"])  # không log email/token

    return {
        "access_token": access_token, "refresh_token": refresh_token,
        "user": { "id": user["id"], "name": user["name"], "email": user["email"],
                  "level": user.get("level"), "streak": user.get("streak", 0),
                  "goal": user.get("goal") }
    }
```

### 3.4 Verify Google `id_token`

```python
from google.oauth2 import id_token as google_id_token
from google.auth.transport import requests as google_requests

GOOGLE_CLIENT_ID = os.getenv("GOOGLE_CLIENT_ID", "")  # KHÔNG hardcode

def _verify_google_id_token(token: str) -> dict:
    """Verify Google id_token. Raise HTTPException on failure."""
    if not GOOGLE_CLIENT_ID:
        raise RuntimeError("GOOGLE_CLIENT_ID not set in .env")
    try:
        payload = google_id_token.verify_oauth2_token(
            token,
            google_requests.Request(),
            GOOGLE_CLIENT_ID,
        )
        return payload
    except ValueError as exc:
        # Token expired, bad audience, bad signature
        raise HTTPException(status_code=400, detail="id_token không hợp lệ hoặc hết hạn.")
    except Exception as exc:
        log.error("Google token verify error: %s", type(exc).__name__)  # không log token
        raise HTTPException(status_code=502, detail="Lỗi xác thực với Google.")
```

### 3.5 Xử lý edge cases

| Edge case | Xử lý |
|---|---|
| Google trả email đã có trong MOCK_USERS (đăng ký email/pw) | Link oauth_account vào user cũ, trả JWT → user đăng nhập được bằng cả 2 cách |
| Google sub đã tồn tại trong OAUTH_ACCOUNTS (đăng nhập lại) | Login trực tiếp, không tạo thêm |
| `password_hash = None` (OAuth user) cố dùng `/api/auth/login` | Sẽ fail tự nhiên do `compare_digest(None)` — không cần xử lý thêm ở S1.6 |
| Email Google không được Google trả về (hiếm) | Fallback: `email = sub + "@google.oauth"`, đủ unique |
| `GOOGLE_CLIENT_ID` chưa set | Log lỗi startup + raise RuntimeError rõ ràng |
| Token hết hạn (exp < now) | google-auth tự check, raise ValueError → 400 |
| Tài khoản suspended | 403 "Tài khoản đã bị khóa." |

### 4.2 authService.js — cập nhật `loginWithGoogle()`

Hiện tại `loginWithGoogle()` dùng redirect sang `${API_URL}/oauth2/authorization/google` (Option B flow cũ, không đúng). Cần **thay hoàn toàn** bằng GSI popup flow:

```javascript
/**
 * Khởi động Google OAuth qua GSI popup.
 * Resolve với response data { access_token, refresh_token, user }.
 * Reject với Error message nếu thất bại.
 */
export function loginWithGoogle() {
  return new Promise((resolve, reject) => {
    if (!window.google?.accounts?.oauth2) {
      reject(new Error("Google Identity Services chưa tải xong. Vui lòng thử lại."));
      return;
    }

    const client = window.google.accounts.oauth2.initTokenClient({
      client_id: import.meta.env.VITE_GOOGLE_CLIENT_ID,
      scope: "openid email profile",
      callback: () => {},  // override bên dưới
    });

    // Dùng id_token flow thay vì access_token flow
    // → initCodeClient / prompt="select_account"
    // Thực tế: dùng google.accounts.id.initialize (One Tap / popup)
    const google = window.google;
    google.accounts.id.initialize({
      client_id: import.meta.env.VITE_GOOGLE_CLIENT_ID,
      callback: async (response) => {
        try {
          const data = await apiClient("/api/auth/google", {
            method: "POST",
            body: JSON.stringify({ id_token: response.credential }),
          });
          resolve(data);
        } catch (err) {
          reject(err);
        }
      },
      auto_select: false,
    });

    google.accounts.id.prompt((notification) => {
      if (notification.isNotDisplayed() || notification.isSkippedMoment()) {
        // Fallback: render button popup
        reject(new Error("Đăng nhập Google bị hủy hoặc không hiển thị. Vui lòng thử lại."));
      }
    });
  });
}
```

> **Lưu ý triển khai:** GSI `id.prompt()` (One Tap) có thể bị chặn do browser settings. Cần có fallback render `renderButton()` ẩn và click nó nếu One Tap không hiển thị. Developer sẽ quyết định UX cụ thể khi implement.

### 4.3 LoginPage.jsx — cập nhật `handleGoogleLogin`

Hiện tại `handleGoogleLogin` chỉ gọi `loginWithGoogle()` (sync, redirect). Cần cập nhật để:
- Gọi `loginWithGoogle()` (async, Promise).
- Xử lý loading state (button disabled khi đang chờ).
- Xử lý success: gọi `login(data)` từ `useAuth`, navigate.
- Xử lý error: hiển thị `setError(err.message)`.

```javascript
const handleGoogleLogin = async () => {
  setError("");
  setGoogleLoading(true);
  try {
    const data = await loginWithGoogle();
    login({ access_token: data.access_token, refresh_token: data.refresh_token });
    navigate(getSafeNext(), { replace: true });
  } catch (err) {
    setError(err.message || "Đăng nhập Google thất bại. Vui lòng thử lại.");
  } finally {
    setGoogleLoading(false);
  }
};
```

Thêm `googleLoading` state riêng (tách khỏi `loading` của email form) để không disable form khi đang chờ Google popup.
