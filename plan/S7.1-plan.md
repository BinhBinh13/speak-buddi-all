## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)

Mục tiêu: phân loại lỗi 502 để FE biết "retry" (Anthropic) hay "degrade text" (ElevenLabs). Hai phương án — chọn **3a** (ít thay đổi, tránh đổi contract `/speak`):

**3a. Tách lỗi qua response thay vì chỉ HTTP status** (`routers/ai.py` — sửa nhẹ):
- Giữ `POST /speak` trả audio khi cả 2 service OK (như hiện tại).
- Khi Claude lỗi → vẫn 502 nhưng body `{"detail": "AI service error", "service": "anthropic"}`.
- Khi Claude OK nhưng TTS lỗi → **KHÔNG fail toàn bộ**: trả **200** dạng JSON `{"reply_text": "...", "audio": null, "tts_error": true}` thay vì 502, để giữ được text reply (đáp ứng "không mất dữ liệu" + fallback text của SRS §5.2). FE phát hiện `Content-Type: application/json` → degrade text-only.
  - *Khác biệt với code hiện tại:* hiện TTS lỗi đang raise 502 và mất luôn text → cần đổi để giữ text.
- Phân biệt response: audio thành công = `StreamingResponse audio/mpeg` (giữ nguyên); TTS lỗi = `JSONResponse`.
- Empty text vẫn giữ 400 `Empty text`.

File đụng tới:
- `routers/ai.py` — sửa nhánh `except TTS` để trả JSON reply-text thay vì 502; thêm field `service` vào detail khi Claude lỗi.
- (tùy chọn) `schemas/ai.py` — thêm `SpeakTextFallbackOut(BaseModel)` cho response degrade.
- Không cần sửa `ai_service.py` / `tts_service.py` / `clients.py` (đã đạt yêu cầu tích hợp + raise).

> Lưu ý §4.5: không log audio/secret — code hiện đã chỉ log `text[:80]`, giữ nguyên.

### Frontend (speak-buddi/)

Thay placeholder bằng màn hội thoại thật, bám mockup `hoi_thoai_ai_desktop`.

- `speak-buddi/src/features/conversation/ConversationPage.jsx` (MỚI) — màn chính: header topic, list chat bubble (user phải / AI trái), vocab panel (từ batch truyền qua `location.state.words`), ô input + nút gửi + nút mic (mic chỉ scaffold, tích hợp ghi âm thuộc Epic 6/sau).
  - Nhận `location.state` từ TopicModal (S2.5): `topicName`, `batchIndex`, `words[]`, `topicId`.
  - State: `messages[]` (`{role, content, audioUrl?, ttsError?}`), `input`, `loading`, `errorBanner`.
  - Lượt đầu: gọi `/speak` với `context: "GREETING_MODE:"` + `topic` để AI chào (đã hỗ trợ ở `ai_service`).
- `speak-buddi/src/features/conversation/services/conversationService.js` (MỚI) — wrap gọi `/speak`:
  - `sendMessage({ text, topic, history })` → POST `/speak`.
  - Xử lý response 2 nhánh: nếu `Content-Type` audio → trả `{ replyText: decodeURIComponent(headers["X-Reply-Text"]), audioUrl: URL.createObjectURL(blob) }`; nếu JSON (TTS fallback) → trả `{ replyText, audioUrl: null, ttsError: true }`; nếu 502 → throw lỗi mang `service: "anthropic"`.
  - **Không dùng `apiClient` chung** vì client.js parse JSON cứng và không đọc được audio blob / header `X-Reply-Text`; viết fetch riêng có gắn JWT từ localStorage (theo pattern `doFetch`).
- `speak-buddi/src/features/conversation/components/ChatBubble.jsx` (MỚI) — bubble + nút play audio + badge "🔇 văn bản thay thế" khi `ttsError`.
- `speak-buddi/src/features/conversation/components/VocabPanel.jsx` (MỚI) — panel từ vựng bên phải (từ `words[]`).
- `speak-buddi/src/app/App.jsx` (SỬA) — route `/conversation` trỏ sang `ConversationPage` thay vì `ConversationPlaceholder` (giữ placeholder file hoặc xóa).

Xử lý lỗi FE (AC-09-04):
- Anthropic 502 → banner đỏ `🔄 AI đang bận, vui lòng thử lại sau vài giây.`, **không xóa** message user vừa gửi (đã push vào `messages`), cho phép nút "Thử lại".
- TTS fallback (200 JSON) → push AI message bình thường + flag `ttsError` → bubble hiển thị text + caption `🔇 Không tạo được âm thanh. Đang hiển thị văn bản thay thế.`
- Input rỗng → disable nút gửi, không gọi API.
