## 1. Mục tiêu & phạm vi

Sau khi Student đăng ký lần đầu (hoặc đăng nhập mà chưa hoàn tất onboarding), hệ thống bắt buộc họ đi qua flow onboarding **4 bước** để thu thập:

| Bước | Câu hỏi | Dữ liệu lưu |
|---|---|---|
| 1 | Trình độ tiếng Anh hiện tại (A1–C2) | `target_level` |
| 2 | Chủ đề muốn học trong level đó (load từ DB) | `interests TEXT[]` (lưu slug/tên topic) |
| 3 | Số phút học mỗi ngày (5 / 10 / 15 / 20) | `daily_minutes` (cột mới) |
| 4 | Số từ mới muốn học mỗi buổi | `words_per_session` (cột mới) |

"Done" = Student mới đăng ký bị điều hướng vào onboarding, phải hoàn tất 4 bước hợp lệ mới vào được app; dữ liệu được lưu xuống `user_profile`; cờ `onboarding_completed = true`; user được đưa ra ngoài. Việc **sinh roadmap thực tế** từ level+topics là phạm vi **S2.2** — không build ở đây.

### 3.1 Backend (`speak-buddi-be/`)

#### Migration DB — **cần chạy trước**
File: `speak-buddi-be/db/migrations/003_onboarding_fields.sql`
```sql
ALTER TABLE user_profile
  ADD COLUMN IF NOT EXISTS daily_minutes   INTEGER NOT NULL DEFAULT 10,
  ADD COLUMN IF NOT EXISTS words_per_session INTEGER NOT NULL DEFAULT 10;
```
Không đổi các cột hiện có; `interests TEXT[]` tái dùng cho selected topics.

#### Repo `repositories/user_repo.py`
Thêm 2 hàm:

**`get_topics_by_level(db, level_code: str) → list[dict]`**
```sql
SELECT t.id, t.name, t.slug
FROM topic t
JOIN level l ON t.level_id = l.id
WHERE l.code = UPPER(:code) AND t.is_active = TRUE
ORDER BY t.display_order, t.name
```

**`update_onboarding(db, user_id, level, topics, daily_minutes, words_per_session)`**
```sql
UPDATE user_profile
SET target_level      = :level,
    interests         = :topics,
    daily_minutes     = :daily_minutes,
    words_per_session = :words_per_session,
    updated_at        = NOW()
WHERE user_id = CAST(:uid AS UUID)
RETURNING target_level, interests, daily_minutes, words_per_session
```
Sau đó `db.commit()`. (`learning_goal` giữ nguyên, không xóa.)

#### Schema `schemas/onboarding.py` (file mới)
```python
class OnboardingRequest(BaseModel):
    level: str
    topics: list[str] = []          # slug hoặc name của topic
    daily_minutes: int = 10
    words_per_session: int = 10

class TopicOut(BaseModel):
    id: str
    name: str
    slug: str

class OnboardingOut(BaseModel):
    level: str
    topics: list[str]
    daily_minutes: int
    words_per_session: int
    onboarding_completed: bool
```

#### Router `routers/onboarding.py` (file mới) — prefix `/api`, đăng ký trong `main.py`

**`GET /api/onboarding/topics?level=A1`** — public hoặc `Depends(current_user)`:
1. Validate `level in {"A1","A2","B1","B2","C1","C2"}` → 400 nếu sai.
2. Gọi `user_repo.get_topics_by_level(db, level)`.
3. Trả `list[TopicOut]`.

**`POST /api/onboarding`** — `Depends(current_user)`:
1. `level = req.level.strip().upper()` → validate BR09 (400 + message §5.2 nếu sai).
2. Validate `req.daily_minutes in {5, 10, 15, 20}` → 400 nếu sai.
3. Validate `req.words_per_session > 0` → 400 nếu sai.
4. Gọi `user_repo.update_onboarding(...)`.
5. Trả `OnboardingOut(onboarding_completed=True)`.
6. Log: `log.info("ONBOARDING ok user=%s level=%s", uid, level)` (không log topics/chi tiết cá nhân).

#### Cập nhật `schemas/auth.py` — `user_to_dict`
Thêm fields:
```python
"onboarding_completed": row["target_level"] is not None,
"daily_minutes":        row.get("daily_minutes", 10),
"words_per_session":    row.get("words_per_session", 10),
```
Cập nhật `_USER_SELECT` để SELECT thêm `daily_minutes`, `words_per_session` từ `user_profile`.
