## 1. Mục tiêu & phạm vi

Story đã được implement với bước 2 = "Chọn chủ đề" (multi-select chip load từ API). **Bản revise này đổi thiết kế product:** bước 2 trở thành **"Mục tiêu học tập"** với **3 lựa chọn cố định** (radio card, không gọi API), và khi submit bước cuối hệ thống **dùng Claude sinh roadmap_sequence cá nhân hóa** từ `level + learning_goal`, lưu vào `user_profile.roadmap_sequence (JSONB)`. GET /api/roadmap (S2.2) đọc sequence này thay vì sort on-the-fly, có fallback backward-compatible.

**Bám USP "Học xong là nói được ngay":** roadmap được tổ chức theo **tình huống giao tiếp thực tế (scenario)**, không phải danh sách từ vựng category. Mỗi node trong sequence mang thêm `scenario_name` (tên tình huống thực tế tiếng Việt do Claude sinh) và `scenario_description` (1 câu — user làm được gì sau node). Vì vậy `roadmap_sequence` lưu **list of objects** giàu thông tin chứ không chỉ list topic_id.

"Done" = onboarding 4 bước chạy với GoalStep, BE lưu `learning_goal` + sinh & lưu `roadmap_sequence` dạng scenario, roadmap render theo thứ tự + scenario AI gợi ý (hoặc fallback cũ nếu NULL), mọi AC dưới đây pass.

## 3. Thiết kế kỹ thuật

### Data model / DB

Cột `learning_goal VARCHAR(255)` ĐÃ tồn tại trong `user_profile` (schema_core.sql dòng 61) — không cần thêm. Chỉ cần thêm `roadmap_sequence`.

Migration mới `speak-buddi-be/db/migrations/012_roadmap_sequence.sql` (idempotent):
```sql
-- Migration 012: thêm roadmap_sequence cho onboarding AI (S2.1 revised v2 — scenario-based)
ALTER TABLE user_profile
  ADD COLUMN IF NOT EXISTS roadmap_sequence JSONB;
-- roadmap_sequence: ordered JSON array of scenario objects, ví dụ:
--   [
--     {"topic_id": "<uuid-greetings>", "scenario_name": "Chào hỏi người mới",
--      "scenario_description": "Tự giới thiệu và bắt đầu cuộc trò chuyện tự nhiên"},
--     {"topic_id": "<uuid-directions>", "scenario_name": "Hỏi đường & Di chuyển",
--      "scenario_description": "Hỏi và chỉ đường khi đi du lịch"}
--   ]
-- NULL = chưa sinh / user cũ → roadmap_repo fallback về sort theo difficulty,
--        scenario_name/scenario_description = None.
```
Lưu **ordered list of scenario objects** (`{topic_id, scenario_name, scenario_description}`). `topic_id` (UUID string) để map sang node; 2 field còn lại do Claude sinh để hiển thị tình huống thực tế. Cập nhật cả khối CREATE TABLE trong `db/schema_core.sql` (thêm dòng `roadmap_sequence JSONB,`) để env mới có sẵn cột. (Số migration cao nhất hiện tại là 011 → dùng 012.)

### Backend (speak-buddi-be/)

**1. `schemas/onboarding.py`** — đổi payload:
- `OnboardingRequest`: bỏ `topics: list[str]`, thêm `learning_goal: str` (slug cố định). Giữ `level`, `daily_minutes`, `words_per_session`.
- `OnboardingOut`: thay `topics` bằng `learning_goal: str`; thêm `roadmap_generated: bool` (để FE biết AI gen có thành công hay fallback). Giữ `onboarding_completed`.
- Giữ `TopicOut` (vẫn dùng nội bộ khi fetch topic cho AI / endpoint /topics) — không xóa.

**2. `routers/onboarding.py`**:
- Thêm `VALID_GOALS = {"travel", "work", "communication"}`; validate `learning_goal` thuộc tập này → 400 nếu sai.
- Endpoint `GET /api/onboarding/topics` FE không còn gọi nhưng GIỮ LẠI (không xóa) để không phá API contract / dùng cho admin/debug. Ghi chú deprecated trong docstring.
- `POST /api/onboarding`: sau khi validate level/goal/minutes/words →
  1. `user_repo.update_onboarding(...)` lưu `target_level`, `learning_goal`, `daily_minutes`, `words_per_session` (bỏ `topics`/`interests`).
  2. Fetch topic theo level: `user_repo.get_topics_by_level(db, level)`.
  3. Gọi `services.roadmap_ai_service.generate_roadmap_sequence(level, learning_goal, topics)` → trả `list[dict]` (`{topic_id, scenario_name, scenario_description}`) đã validate (subset & đúng thứ tự).
  4. Nếu list không rỗng → `user_repo.update_roadmap_sequence(db, user_id, sequence)`; set `roadmap_generated=True`. Nếu AI lỗi/empty → KHÔNG raise, để `roadmap_sequence=NULL`, `roadmap_generated=False` (fallback ở GET /api/roadmap lo). Degrade an toàn — onboarding vẫn coi như hoàn tất (AC-03-02 lưu level+goal vẫn pass; roadmap fallback vẫn hiển thị).
  5. Log `ONBOARDING ok user=... level=... goal=... roadmap_n=...` (không log nội dung cá nhân chi tiết / scenario text — SRS §4.5).

**3. `services/roadmap_ai_service.py` (MỚI)** — tái dùng pattern `ai_service.py` (`get_claude_client()`, `ANTHROPIC_MODEL`):
- Hàm `generate_roadmap_sequence(level: str, goal: str, topics: list[dict]) -> list[dict]` — trả list of `{topic_id, scenario_name, scenario_description}`.
- `topics` là list `{id, name, slug}` từ `get_topics_by_level`. Nếu `len(topics) <= 1` → return `[{"topic_id": t["id"], "scenario_name": None, "scenario_description": None} for t in topics]` (không cần gọi AI; có thể vẫn gọi để có scenario nếu muốn, nhưng tối thiểu phủ đủ).
- Build prompt (template dưới), gọi Claude qua `client.messages.create(model=ANTHROPIC_MODEL, max_tokens=1500, system=..., messages=[...])`, yêu cầu model trả JSON array of objects thuần (không markdown fence).
- **Parse:** `response.content[0].text.strip()`, strip code-fence nếu có, `json.loads` → list.
  - Nếu mỗi element là **string** (format cũ / caching cũ) → convert `{"topic_id": s, "scenario_name": None, "scenario_description": None}` để backward-compat.
  - Nếu mỗi element là **object** → giữ `topic_id`, `scenario_name`, `scenario_description`.
- **Anti-hallucination:** validate `topic_id` trong mỗi object ∈ tập id đã fetch; loại object có `topic_id` lạ hoặc trùng; append các topic_id model bỏ sót vào cuối (giữ thứ tự difficulty/order gốc) với `scenario_name=None, scenario_description=None` để sequence luôn phủ đủ topic. Parse fail → return `[]` (caller fallback). Bọc try/except, log warning, KHÔNG để exception nổi lên router.
- Map goal slug → mô tả tiếng Anh: `travel`→"travel and tourism", `work`→"work and professional/business contexts", `communication`→"everyday daily communication".

**4. `repositories/user_repo.py`**:
- Sửa `update_onboarding(...)`: bỏ tham số `topics`, thêm `learning_goal: str`; SET `target_level`, `learning_goal`, `daily_minutes`, `words_per_session`; `RETURNING target_level, learning_goal, daily_minutes, words_per_session`. Set `interests = NULL` để tránh dữ liệu lệch (interests không còn drive roadmap).
- Thêm `update_roadmap_sequence(db, user_id, sequence: list[dict]) -> bool`: `UPDATE user_profile SET roadmap_sequence = CAST(:seq AS JSONB), updated_at = NOW() WHERE user_id = CAST(:uid AS UUID) RETURNING user_id`. Tham số `seq = json.dumps(sequence)`; `await db.commit()`.

**5. `repositories/roadmap_repo.py`**:
- `get_profile_for_roadmap`: thêm `p.roadmap_sequence` vào SELECT.
- `get_roadmap`: sau khi có `topics` từ `get_roadmap_topics`, nếu `roadmap_sequence` không NULL/rỗng → **reorder** topics theo sequence:
  - Parse từng element của sequence: `{"topic_id": ..., "scenario_name": ..., "scenario_description": ...}` (element string format cũ → coi như `{"topic_id": s, "scenario_name": None, "scenario_description": None}`).
  - `by_id = {t["id"]: t for t in topics}`.
  - Duyệt sequence theo thứ tự: với mỗi item có `topic_id in by_id` → build node từ `by_id[topic_id]`, đồng thời gán thêm `"scenario_name"` và `"scenario_description"` vào dict node từ item sequence.
  - Append topic còn lại không nằm trong sequence (topic mới thêm sau onboarding) vào cuối, giữ thứ tự gốc, với `scenario_name = None`, `scenario_description = None`.
  - Gán `order_index = idx` theo thứ tự mới khi build nodes.
  - Nếu `roadmap_sequence` NULL/rỗng → giữ behavior cũ (ORDER BY difficulty đã có trong `get_roadmap_topics`); mỗi node `scenario_name = None`, `scenario_description = None`. KHÔNG đổi SQL ORDER BY → fallback tự nhiên.
- `is_interest`: bước 2 không còn set interests → luôn false. Giữ field để không phá schema RoadmapNode/S2.4; `selected_topics` = 0 (chấp nhận). Không xóa logic interest để tránh hồi quy.

**6. `schemas/roadmap.py` — `RoadmapNode`**:
- Thêm 2 field optional:
  ```python
  scenario_name: str | None = None          # tên tình huống giao tiếp do AI sinh
  scenario_description: str | None = None   # user làm được gì sau node này
  ```
- Backward-compat: `None` khi `roadmap_sequence` NULL (user cũ / fallback) hoặc node được append ngoài sequence.

**7. `routers/roadmap.py`** — không đổi logic (vẫn nhận dict từ repo + map RoadmapOut); nhờ Pydantic tự pick `scenario_name`/`scenario_description` từ dict node. Verify không vỡ contract.

**8. `main.py`** — không thêm router mới (service gọi nội bộ từ onboarding router). Không đổi.

#### Prompt template cho Claude (roadmap_ai_service)

System prompt:
```
You are a curriculum designer for a Vietnamese English-learning app.
The app's USP is: "Learn English and speak it immediately in real life."
Your task: given a learner's CEFR level and learning goal, reorder the available topics
into a scenario-based learning path — each topic becomes a real-life communication scenario.

Rules:
- Use ONLY the topic_ids provided. Do NOT invent ids.
- Include every topic_id exactly once.
- For each topic, assign a Vietnamese scenario_name (a real-life situation, e.g. "Đặt phòng khách sạn") — NOT the raw topic category name.
- Write a short Vietnamese scenario_description (1 sentence): what the learner will be able to DO after completing this node.
- Order: most immediately useful/relevant to the goal first; foundational language first within same relevance.
- Reply with a raw JSON array ONLY. No prose, no markdown fences.
```
User message (ví dụ):
```
CEFR level: A1
Learning goal: travel and tourism
Topics:
- topic_id=<uuid1> name="Greetings"
- topic_id=<uuid2> name="Numbers"
- topic_id=<uuid3> name="Food & Dining"
- topic_id=<uuid4> name="Hotels"
- topic_id=<uuid5> name="Directions"

Return ordered JSON array:
[{"topic_id":"...","scenario_name":"...","scenario_description":"..."},...]
```
`max_tokens` đủ lớn (vd 1500) vì mỗi object có thêm 2 chuỗi tiếng Việt. Parse `response.content[0].text.strip()`, strip code-fence nếu có, `json.loads`, rồi validate như mục 3 (BE-3).
