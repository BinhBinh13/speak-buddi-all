- **Ngày lập:** 2026-06-07

## 1. Mục tiêu & phạm vi

Bổ sung hành động **Disable (vô hiệu hóa)** cho Admin trên ba loại nội dung đã có từ S9.1: **Topic**, **Vocabulary (topic_word)**, **Vocabulary Test**. Admin bấm Disable → **modal xác nhận** → BE set `is_active = FALSE` (AC-13-03). Nội dung bị disable **không còn hiển thị** cho học viên (API read-side đã lọc `is_active=TRUE` từ S3.x/S4.x) nhưng **vẫn giữ row DB** và tiến độ học cũ (S3.4 — progress không bị xóa).

"Done" = Admin disable được topic/word/test qua UI + API; sau disable, student không thấy nội dung đó ở `/vocabulary`, `/quiz`, `/roadmap`; Admin vẫn thấy bản ghi inactive khi filter Status = Inactive/All.

> **Phạm vi loại trừ:**
> - **Re-enable (`is_active=true`)** — AC-13-03 chỉ nói disable. Khôi phục nội dung crawl/admin → **S9.5** (AC-13-06). S9.2 không làm nút "Kích hoạt lại" trừ khi trivial (PUT set is_active) — ghi nhận TBD, ưu tiên **chỉ disable**.
> - **Hard delete** — không có trong SRS §5.3.
> - **Crawler Langeek** — S9.3/S9.4.
> - **Payment plan soft delete** — S10.2 (cùng pattern, story riêng).

### Backend (speak-buddi-be/)

**Repository — bổ sung `repositories/content_repo.py`:**
```python
async def soft_delete_topic(db, topic_id: str) -> bool   # True nếu row tồn tại (kể cả đã inactive)
async def soft_delete_word(db, word_id: str) -> bool
```
- SQL: `UPDATE ... SET is_active = FALSE WHERE id = :id` (idempotent — gọi lại vẫn FALSE).
- Trả `False` nếu 0 row affected (404 ở router).

**Repository — tái dùng `repositories/quiz_repo.py`:**
- `soft_delete_test(db, test_id)` — đã có; có thể bổ sung return bool/check row tồn tại nếu cần 404 chính xác.

**Router — mở rộng `routers/admin_content.py`** (giữ `dependencies=[Depends(require_admin)]`):

| Method | Path | Mô tả | Response |
|---|---|---|---|
| DELETE | `/api/admin/topics/{topic_id}` | Soft-delete topic | 204 / 404 |
| DELETE | `/api/admin/words/{word_id}` | Soft-delete word | 204 / 404 |
| DELETE | `/api/admin/tests/{test_id}` | Soft-delete test | 204 / 404 |

> Dùng **DELETE** semantic cho "disable" (không hard-delete) — body rỗng; log `ADMIN_DISABLE_*` với admin `sub`. Không yêu cầu body confirm (confirm ở FE); AC "when action is confirmed" = modal FE.

**Query param list (đã có — wire FE filter):**
- `GET /api/admin/topics?include_inactive=true|false` — mặc định `false` (chỉ active); Admin chọn "All" → `true`.
- `GET /api/admin/words?include_inactive=...` — tương tự.
- `GET /api/admin/tests` — mặc định `include_inactive=true` (S9.1); thêm filter FE Active-only khi cần.

**Không sửa schema.** Không cascade disable con (topic disable **không** auto-disable words/tests — SRS không yêu cầu cascade; word/test inactive độc lập. Ghi chú trong verify: topic inactive → learner không vào topic đó; words/tests orphan vẫn ẩn qua filter riêng).
