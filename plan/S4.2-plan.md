## 1. Tổng quan

### Mục tiêu
Triển khai màn hình làm bài kiểm tra từ vựng với 4 dạng câu hỏi:
- **flashcard** — hiện từ, user flip xem nghĩa rồi tự chấm "Biết / Không biết"
- **multiple_choice** — hiện câu hỏi + 4 lựa chọn, chọn 1
- **fill_blank** — hiện câu thiếu từ, user gõ vào ô input (có word bank gợi ý)
- **grammar_mapping** — kéo/click nối từ với nghĩa tương ứng

### Phạm vi S4.2
- **Backend:** 2 router endpoint mới (`GET /api/tests/{test_id}`, `GET /api/tests/{test_id}/questions`) + 1 endpoint danh sách test theo topic (`GET /api/tests?topic_id=`)
- **Frontend:** Route `/quiz/:testId`, trang `QuizPage` chứa `QuizRunner` điều phối 4 loại câu hỏi
- **Không** bao gồm: logic tạo attempt, submit, chấm điểm, lưu kết quả (→ S4.3 + S4.4)

### Phạm vi ngoài S4.2
- Nút "Làm bài kiểm tra" trên `VocabularyPage` hiện đang `disabled` placeholder → S4.2 sẽ bật nút này.
- `generate_questions_from_topic_words()` trong `quiz_service.py` hiện là placeholder → S4.2 sẽ implement logic auto-generate.

## 2. Phân tích Acceptance Criteria

### AC-06-01 — Flashcard
- Hiện từ (mặt trước): `question_text` to + phonetic (nếu có qua `topic_word_id`)
- User click/tap để lật thẻ (3D flip animation theo mockup)
- Mặt sau: nghĩa tiếng Việt + ví dụ
- Sau flip: hiện 2 nút **"Không biết"** (tertiary/outline) và **"Đã biết"** (primary)
- Edge case: flashcard không gắn `topic_word_id` → không hiển thị phonetic, hiển thị `question_text` đơn giản

### AC-06-01 — Multiple Choice
- Câu hỏi dạng text, 4 lựa chọn từ `answers[]`
- Khi chọn 1 option: highlight border primary, radio icon filled
- Chỉ có thể chọn 1 lựa chọn (radio behavior)
- Button "Tiếp theo" active sau khi đã chọn
- Edge case: ít hơn 4 answers (dữ liệu thiếu) → vẫn hiển thị bình thường, không crash

### AC-06-01 — Fill Blank
- Câu có `___` placeholder trong `question_text` → tách thành prefix + blank + suffix
- Input text ở giữa câu
- Word bank (helper): các từ từ `answers[]` (distractor + đúng) để user click chọn nhanh
- Button "Kiểm tra" disabled khi input rỗng, active khi có text
- Validate client-side (case-insensitive, trim) trước khi tiếp theo
- Edge case: `question_text` không chứa `___` → hiển thị toàn bộ làm label, input thêm vào cuối

### AC-06-01 — Grammar Mapping
- 2 cột: cột trái (từ/cụm từ nguồn), cột phải (nghĩa/cụm đích)
- Thứ tự cột phải bị shuffle trước khi hiển thị
- Click-to-match (mobile-friendly): click item trái → highlight, click item phải → nối pair
- Hiển thị đường nối hoặc badge màu để biểu thị cặp đã nối
- Nút "Xóa kết nối" trên mỗi cặp đã nối
- Nút "Kiểm tra" active khi tất cả items bên trái đã được nối
- Edge case: số item trái ≠ số item phải (dữ liệu lỗi) → cảnh báo, không crash
- **Kỹ thuật:** Implement bằng click-to-match thay vì HTML5 drag-drop thuần để đảm bảo mobile touch (dùng drag-drop với fallback click trên desktop là plus)

### Edge cases chung
- Test không có câu hỏi nào → hiển thị empty state
- API lỗi khi load questions → hiển thị error + retry
- User navigate back giữa chừng → không mất trạng thái (state local trong component)
- Câu hỏi không thuộc 4 loại trên → bỏ qua, log warning

## 3. Thiết kế Backend

### 3.1 Endpoints mới

#### `GET /api/tests?topic_id={uuid}`
Lấy danh sách test active của 1 topic.

```
Response 200:
[
  {
    "id": "uuid",
    "topic_id": "uuid",
    "level_id": "uuid",
    "title": "string",
    "description": "string|null",
    "is_active": true,
    "created_by": "uuid|null",
    "created_at": "datetime"
  }
]
```

Sử dụng `quiz_repo.get_tests_by_topic()` đã có.

#### `GET /api/tests/{test_id}`
Lấy metadata 1 bài kiểm tra.

```
Response 200: VocabularyTestOut
Response 404: {"detail": "Test not found or inactive"}
```

#### `GET /api/tests/{test_id}/questions`
Lấy toàn bộ câu hỏi + đáp án của bài kiểm tra.

```
Response 200:
[
  {
    "id": "uuid",
    "vocabulary_test_id": "uuid",
    "topic_word_id": "uuid|null",
    "question_text": "string",
    "question_type": "flashcard|multiple_choice|fill_blank|grammar_mapping",
    "display_order": 0,
    "answers": [
      {
        "id": "uuid",
        "quiz_question_id": "uuid",
        "answer_text": "string",
        "is_correct": true,
        "display_order": 0
      }
    ]
  }
]
Response 404: {"detail": "Test not found"}
```

**Lưu ý bảo mật:** Với `multiple_choice`, `grammar_mapping`, và `fill_blank` — backend trả về `is_correct` trong `answers`. Điều này acceptable cho MVP vì quiz không phải exam nghiêm ngặt; S4.3 sẽ validate server-side khi submit. Nếu muốn ẩn đáp án: tạo schema riêng `QuizQuestionForUserOut` omit `is_correct`.

### 3.2 Router mới: `routers/quiz.py`

```python
# Pattern bám routers/learning.py
router = APIRouter(prefix="/api", tags=["quiz"])

@router.get("/tests", response_model=list[VocabularyTestOut])
async def list_tests_by_topic(topic_id: uuid.UUID, ...)

@router.get("/tests/{test_id}", response_model=VocabularyTestOut)
async def get_test(test_id: uuid.UUID, ...)

@router.get("/tests/{test_id}/questions", response_model=list[QuizQuestionOut])
async def get_test_questions(test_id: uuid.UUID, ...)
```

Đăng ký trong `main.py`:
```python
from routers import ai, auth, learning, quiz
app.include_router(quiz.router)
```

### 3.3 Service: `quiz_service.generate_questions_from_topic_words()`
Implement logic placeholder hiện có — tạo tập câu hỏi tự động từ danh sách từ vựng:

```
Input: list[dict] — topic_words với {id, word, meaning_vi, phonetic, example_sentence, ...}
Output: list[dict] — câu hỏi chưa lưu DB

Logic:
- Mỗi từ → 1 flashcard question
- Với N >= 4 từ → tạo multiple_choice cho mỗi từ (distractor = 3 từ random khác trong danh sách)
- Với N >= 4 từ → tạo 1 grammar_mapping question gộp 4 từ đầu
- fill_blank: chỉ tạo nếu từ có example_sentence chứa chính từ đó (thay thế bằng ___)
```

Hàm này dùng nội bộ, chưa expose endpoint (endpoint create sẽ ở S9.1).

### 3.4 Schemas bổ sung (minimal)

Schemas đã có trong `schemas/quiz.py` đủ dùng cho S4.2. Không cần thêm schema mới.

### 4.5 UI logic chi tiết

#### FlashcardQuestion
- **3D flip**: dùng CSS `perspective`, `transform-style: preserve-3d`, `rotateY(180deg)` bám mockup `bai_kiem_tra_flashcard_desktop`
- Mặt trước: từ (display font lớn) + phonetic nhỏ + icon audio (nút play, gọi TTS nếu có)
- Mặt sau: nghĩa tiếng Việt (h1 primary) + ví dụ (body-md) + dịch
- Khi chưa flip: nút "Tap để lật"
- Sau flip: nút "Không biết" (outline tertiary) và "Đã biết" (primary)
- Cả hai nút gọi `onAnswer(selfRating)` → QuizRunner ghi vào `userAnswers` và nextQuestion

#### MultipleChoiceQuestion
- Grid 2 cột (desktop), 1 cột (mobile)
- Mỗi option: card có radio hidden, label click được, bám mockup `bai_kiem_tra_desktop`
- Selected state: `border-primary`, `bg-primary-container/10`, icon `radio_button_checked`
- Default state: `border-outline-variant`, icon `radio_button_unchecked`
- "Tiếp theo" button: disabled khi `selectedId === null`
- onClick "Tiếp theo": gọi `onAnswer({answerId: selectedId})` → next

#### FillBlankQuestion
- Tách `question_text` theo marker `___` → `[prefix, suffix]`
- Nếu không có `___`: prefix = question_text, suffix = ""
- Input: inline trong câu, border-bottom chỉ (không box), focus → `border-primary`
- Word bank: các `answer_text` từ `answers[]` (gộp đúng + sai), hiển thị chip button
- Click chip → điền vào input, highlight chip đã chọn (ring-2 primary)
- "Kiểm tra" button: disabled khi input rỗng, active khi có text
- onClick: validate client-side bằng `check_fill_blank_answer` logic (normalize + casefold), gọi `onAnswer({textAnswer: inputValue})`
- Không hiện đúng/sai ngay (defer đến S4.4 — trang kết quả)

#### GrammarMappingQuestion
- Lấy `answers[]` → chia thành: left (chuỗi trước `→`) và right (chuỗi sau `→`)
  - `answer_text` format: `"hello → xin chào"` → split bằng ` → `
- Shuffle right column trước render (dùng Fisher-Yates, 1 lần khi mount)
- **Click-to-match**:
  1. Click item trái → `selectedLeft = index`, highlight viền primary
  2. Click item phải khi đã có `selectedLeft` → tạo pair, bỏ highlight
  3. Click item đã nối → xóa pair tương ứng
  4. Click item trái khi đã có `selectedLeft` khác → chuyển selection
- Hiển thị pairs: badge màu số (1,2,3,4) cạnh item đã nối (cả trái và phải của cùng cặp dùng cùng màu/số)
- "Kiểm tra": active khi tất cả items trái đã nối (pairs.length === leftItems.length)
- onClick: gọi `onAnswer({mappingPairs: pairs})`

## 7. Rủi ro & Phụ thuộc

### Rủi ro

| Rủi ro | Mức độ | Giảm thiểu |
|---|---|---|
| Grammar mapping: UX drag-drop phức tạp trên mobile | Cao | Dùng click-to-match thuần (không cần HTML5 DnD). Nếu cần DnD desktop: thêm sau. |
| `is_correct` lộ ra client → user có thể gian lận | Thấp (MVP) | Acceptable cho học tập; S4.3 validate server-side khi submit |

| 3D flip CSS: browser cũ không hỗ trợ `transform-style: preserve-3d` | Thấp | Graceful fallback: nếu không hỗ trợ → flip bằng opacity thay vì 3D |
| API `GET /api/tests/:testId/questions` trả list rất lớn (> 100 câu) | Thấp | MVP không phân trang; đủ dùng cho các bài test thực tế (10–30 câu) |
