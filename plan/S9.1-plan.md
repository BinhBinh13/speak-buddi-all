## 1. Mục tiêu & phạm vi

Cung cấp giao diện Admin để **tạo / sửa** ba loại nội dung học: **Topic (chủ đề)**, **Vocabulary (topic_word)**, và **Vocabulary Test** (kèm question/answer). Mỗi form khi submit hợp lệ thì lưu (AC-13-01); khi dữ liệu sai (trùng tên topic, thiếu trường bắt buộc...) thì hiển thị **lỗi validation inline ngay tại field** (AC-13-02), đúng chuẩn message §5.2.

"Done" = Admin (role `admin`) đăng nhập, vào khu vực Admin, có thể CRUD (Create/Read/Update; **soft delete tách sang S9.2**) topic / word / test thông qua API thật trên backend, có validation cả FE (inline) lẫn BE (nguồn sự thật chống trùng). Guest/student bị chặn khỏi route + API admin.

> **Phạm vi loại trừ rõ ràng:**
> - **Soft delete `is_active=false` → S9.2** (story này chỉ tạo/sửa; vẫn để cột `is_active` trong schema sẵn).
> - **Crawler Langeek → S9.3/S9.4.**
> - **Làm bài quiz / chấm điểm phía học viên → Epic 4** (story này chỉ là CRUD nội dung test cho Admin).

## 3. Thiết kế kỹ thuật

### Data model / DB (tiền đề bắt buộc — xem Rủi ro §6)

S9.1 thao tác trên các bảng `level`, `topic`, `topic_word`, `tag`, `vocabulary_test`, `quiz_question`, `quiz_answer`. Trong `speak-buddi-be/db/schema_core.sql` **hiện chỉ có** nhóm Identity/Auth + Payment — **chưa có** các bảng learning content/quiz (đúng theo chú thích: thuộc S3.1, S4.1). Các bảng này được định nghĩa đầy đủ ở `speak-buddi-docs/db/schema-draft.sql` (bản MySQL tham chiếu) và phải được port sang **PostgreSQL**.

- **Đề xuất:** tạo file mới `speak-buddi-be/db/schema_content.sql` port các bảng từ schema-draft (PostgreSQL syntax: `BIGSERIAL`/`UUID`, `BOOLEAN`, `TEXT[]`/`JSONB`, `TIMESTAMPTZ`, trigger `set_updated_at` tái dùng từ schema_core). Các bảng cần cho S9.1: `level`, `topic`, `tag`, `topic_word`, `topic_word_tag`, `vocabulary_test`, `quiz_question`, `quiz_answer`.
- Cột `is_active BOOLEAN NOT NULL DEFAULT TRUE` giữ sẵn (dùng ở S9.2). List API mặc định chỉ trả `is_active=true` nhưng cho phép param `include_inactive` cho Admin.
- Ràng buộc chống trùng (phục vụ AC-13-02): `UNIQUE(topic.name)` + `UNIQUE(topic.slug)`; `UNIQUE(topic_id, word)` trên `topic_word`.
- Index: `idx_topic_level`, `idx_topic_word_topic`, `idx_vocabulary_test_topic/level`.
- Seed tối thiểu bảng `level` (A1, A2, B1, B2, C1, C2) để form có dropdown.

> ⚠️ Nếu chủ trương dự án là **không tự ý mở rộng schema trước S3.1/S4.1**, story này **blocked** chờ S3.1 (topic/word) và S4.1 (test/question/answer). Xem §6.

### Backend (speak-buddi-be/)

Hiện `main.py` là một file phẳng, auth dùng `MOCK_USERS` RAM, `db/connection.py` còn ở chế độ `_DummyEngine` (DB chưa bật). Để làm S9.1 sạch:

1. **Bật kết nối DB thật** — uncomment phần SQLAlchemy async trong `db/connection.py`, set `DATABASE_URL` (PostgreSQL). Cung cấp `get_db()` Depends thật.
2. **Phân quyền admin** — thêm dependency `require_admin(user = Depends(current_user))` trong `main.py` (hoặc tách `auth_deps.py`): đọc claim `role` từ JWT; nếu khác `admin` → 403. *(Hiện JWT chưa có claim `role` và MOCK_USERS chưa có admin → xem §6; cần `_make_access` thêm `role`, và có ít nhất 1 user admin.)*
3. **Tách router admin content** — file mới `speak-buddi-be/routers/admin_content.py` (FastAPI `APIRouter(prefix="/api/admin", dependencies=[Depends(require_admin)])`), `app.include_router(...)` trong `main.py`.

**Pydantic schemas** (trong router hoặc `schemas/content.py`):
- `TopicCreate` / `TopicUpdate` / `TopicOut` — `name`, `slug`, `level_id?`, `description?`, `display_order`.
- `WordCreate` / `WordUpdate` / `WordOut` — `topic_id`, `level_id?`, `word`, `phonetic?`, `meaning_vi`, `meaning_en?`, `example_sentence?`, `tag_ids?`.
- `TestCreate` / `TestUpdate` / `TestOut` — `title`, `topic_id?`, `level_id?`, `description?`.
- `QuestionIn` (`question_text`, `question_type`, `answers: list[AnswerIn]`), `AnswerIn` (`answer_text`, `is_correct`). Test có thể nhận nested questions để khớp màn editor master-detail.

**Endpoints (CRUD, soft delete hoãn S9.2):**

| Method | Path | Mô tả |
|---|---|---|
| GET | `/api/admin/levels` | list level cho dropdown |
| GET | `/api/admin/topics` | list (search, filter level, paginate) |
| POST | `/api/admin/topics` | tạo topic — validate trùng `name`/`slug` |
| PUT | `/api/admin/topics/{id}` | sửa topic |
| GET | `/api/admin/words` | list word (search, filter topic/level, paginate) |
| POST | `/api/admin/words` | tạo word — validate trùng `(topic_id, word)` |
| PUT | `/api/admin/words/{id}` | sửa word |
| GET | `/api/admin/tests` | list test |
| GET | `/api/admin/tests/{id}` | chi tiết test + questions + answers |
| POST | `/api/admin/tests` | tạo test (+nested questions/answers) |
| PUT | `/api/admin/tests/{id}` | sửa test + sync questions/answers |

**Validation BE (nguồn sự thật cho AC-13-02):**
- Trường bắt buộc rỗng → 400 `Vui lòng điền đầy đủ thông tin.`
- Trùng tên topic / từ trùng trong topic → 409 với `detail` rõ field (vd `Tên chủ đề đã tồn tại.`). FE map vào field tương ứng.
- Test phải có ≥1 question và mỗi question (MC) phải có ≥1 đáp án đúng → 400 (tận dụng cảnh báo "Missing Correct Answer" trong mockup).
- Dùng `_validate_*` helper thuần như pattern sẵn có trong `main.py`.
