# Plan — S7.2: Quota free 15p/5h + reset → 429

- **Epic:** EPIC-7 — Hội thoại AI (UC09 · F09, F10, F11 · BR02–BR05)
- **Ưu tiên:** P1
- **AC nguồn:** AC-09-01, AC-09-02 · BR02, BR03 · FR-10 (kèm liên quan AC-09-03/BR05 — để S7.3 hoàn thiện)
- **Ngày lập:** 2026-06-07

## 1. Mục tiêu & phạm vi
Áp hạn mức (quota) hội thoại AI cho **free Student**: tối đa **15 phút (900 giây) trong mỗi cửa sổ trượt/cố định 5 giờ**, tự reset sau 5 giờ. Khi user còn quota → cho phép hội thoại và **cộng dồn thời gian đã dùng** (AC-09-01). Khi đã dùng hết 15 phút trong cửa sổ hiện tại → backend trả **HTTP 429** kèm thông tin reset, FE hiển thị banner cảnh báo + prompt nâng cấp/chờ (AC-09-02).

"Done" = (1) backend có bảng + helper quota, (2) endpoint hội thoại (`/speak`) bị guard quota cho free user và cộng thời gian, (3) có endpoint đọc quota status cho FE, (4) FE hiển thị quota còn lại + xử lý 429 đúng message SRS. Paid user **không** bị áp quota (chuẩn bị nền cho S7.3).

## 2. Acceptance Criteria cần thỏa
- [ ] **AC-09-01:** Free Student còn quota → start/continue hội thoại được, hệ thống đếm thời gian sử dụng (cộng `used_seconds` vào cửa sổ hiện tại).
- [ ] **AC-09-02:** Free Student đã dùng đủ 15 phút trong cửa sổ 5h hiện tại → request trả `429` + payload báo quota hết + thời điểm reset; FE hiển thị message SRS và prompt nâng cấp/chờ.
- [ ] **BR02:** Giới hạn đúng 15 phút / 5 giờ (max_seconds = 900, window = 5h).
- [ ] **BR03:** Sau 5 giờ, cửa sổ cũ hết hạn → tạo cửa sổ mới, quota reset về 0 (auto, không cần cron — lazy reset lúc check).
- [ ] (liên quan) **AC-09-03/BR05:** Paid user bỏ qua toàn bộ logic quota (kiểm tra `is_paid` từ JWT).

## 3. Thiết kế kỹ thuật

### Data model / DB
File migration mới: `speak-buddi-be/db/migrations/007_ai_quota_window.sql` (PostgreSQL, idempotent `IF NOT EXISTS`, theo phong cách migration 006). Đồng thời append định nghĩa tương ứng vào `speak-buddi-be/db/schema_progress.sql` để khớp quy ước "append cuối file" của repo.

> Lưu ý: `speak-buddi-docs/db/schema-draft.sql` viết theo **MySQL** (`BIGINT UNSIGNED`, `ENGINE=InnoDB`). Dự án thực chạy **PostgreSQL** (xác nhận trong MEMORY + migration 006). Port sang Postgres: UUID PK + `gen_random_uuid()`, `TIMESTAMPTZ`, FK `users(id) ON DELETE CASCADE`.

Bảng `ai_quota_window` (port từ schema-draft dòng 418–433):
```sql
CREATE TABLE IF NOT EXISTS ai_quota_window (
    id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID        NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    window_start_at TIMESTAMPTZ NOT NULL,
    window_end_at   TIMESTAMPTZ NOT NULL,          -- = window_start_at + 5h
    used_seconds    INTEGER     NOT NULL DEFAULT 0 CHECK (used_seconds >= 0),
    max_seconds     INTEGER     NOT NULL DEFAULT 900,  -- BR02: 15 phút
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, window_start_at)
);
CREATE INDEX IF NOT EXISTS idx_ai_quota_user_end ON ai_quota_window (user_id, window_end_at);
```
- Index `(user_id, window_end_at)` để tìm nhanh cửa sổ đang active của user.
- Không tạo bảng `speaking_session` ở story này (thuộc S6.x/S7.1); chỉ làm phần tối thiểu cho quota. Ghi rõ trong implement log nếu S7.1 đã có bảng đó để tránh trùng.
- Trigger `set_updated_at` (dùng lại hàm có sẵn trong `schema_core.sql`, theo pattern migration 005/progress).

### Backend (speak-buddi-be/)
**1. Repository** `speak-buddi-be/repositories/quota_repo.py` (mới — raw SQL `text()` async giống `session_repo.py`):
- `get_or_create_active_window(db, user_id, window_seconds=18000, max_seconds=900) -> dict`
  - Tìm row của user có `window_end_at > NOW()`. Nếu có → trả về (đây là lazy-reset BR03: cửa sổ cũ hết hạn tự bị bỏ qua, tạo mới).
  - Nếu không → INSERT window mới `window_start_at=NOW()`, `window_end_at=NOW()+5h`, `used_seconds=0`.
- `add_used_seconds(db, window_id, seconds) -> dict` — `UPDATE ... SET used_seconds = used_seconds + :sec, updated_at=NOW() RETURNING ...`.
- (tuỳ chọn) `get_quota_status(db, user_id)` — gọi `get_or_create_active_window` rồi tính `remaining_seconds`, `reset_at`.

**2. Helper / dependency** `speak-buddi-be/auth/deps.py` (thêm) hoặc service riêng `services/quota_service.py`:
- `enforce_ai_quota(user, db)`:
  - Nếu `user.get("is_paid")` → return `None` (paid bỏ qua — AC-09-03/BR05).
  - Lấy active window. Nếu `used_seconds >= max_seconds` → `raise HTTPException(429, detail={...})` với payload:
    ```json
    { "code": "quota_exceeded",
      "message": "⏱ Bạn đã dùng hết 15 phút luyện nói. Vui lòng quay lại sau khi quota được reset hoặc nâng lên Pro để luyện không giới hạn.",
      "reset_at": "<window_end_at ISO>",
      "used_seconds": 900, "max_seconds": 900 }
    ```
  - Ngược lại return window dict để route cộng thời gian sau khi xử lý xong.
- Message phải đúng nguyên văn SRS §5.2 (dòng "Free quota exceeded").

**3. Router** `speak-buddi-be/routers/ai.py` (sửa) — guard endpoint `/speak`:
- Thêm `Depends(current_user)` + `Depends(get_db)` vào `/speak` (hiện tại `/speak` chưa auth → cần thêm để biết user/role; xác nhận với S7.1 nếu đã đổi).
- Trước khi gọi AI: `window = enforce_ai_quota(user, db)`.
- Sau khi reply thành công: nếu không phải paid → `add_used_seconds(db, window["id"], delta)`.
  - **Cách tính delta:** ưu tiên cho FE gửi `elapsed_seconds` trong `SpeakRequest` (thời lượng turn người dùng đã ở trong session kể từ lần ping trước), backend cộng dồn. Nếu chưa có cơ chế đo → fallback cộng 1 lượng cố định/heartbeat. Quyết định chốt khi implement S7.1 (xem Rủi ro).
- Cập nhật `schemas/ai.py` (`SpeakRequest`) thêm field optional `elapsed_seconds: int = 0` (hoặc cơ chế heartbeat).

**4. Endpoint quota status** trong `routers/ai.py` (mới):
- `GET /api/ai/quota` → `Depends(current_user)` → trả `{ is_paid, used_seconds, max_seconds, remaining_seconds, reset_at }`. Paid trả `is_paid=true, unlimited=true`. FE dùng để render thanh quota khi mở màn hội thoại.
- (tuỳ chọn) `POST /api/ai/quota/consume` nhận `seconds` để FE chủ động heartbeat đếm thời gian thực, tách khỏi `/speak` — sạch hơn cho việc đếm "thời gian ở trong session" thay vì đếm theo số lần gọi AI. Cân nhắc dùng cách này để AC-09-01 ("counts usage time") chính xác hơn.

### Frontend (speak-buddi/)
- **Service** `speak-buddi/src/features/conversation/services/quotaService.js` (mới):
  - `getQuota()` → `GET /api/ai/quota`.
  - `consumeQuota(seconds)` → nếu chọn endpoint heartbeat.
- **apiClient (sửa)** `speak-buddi/src/shared/api/client.js`:
  - Hiện `apiClient` chỉ throw `Error(message)`, **mất status code** → FE không phân biệt được 429. Cần gắn `err.status = res.status` (và `err.body`) vào Error trước khi throw, hoặc cho `quotaService` tự `fetch` để đọc `res.status === 429`. **Đây là thay đổi load-bearing** — bắt buộc để phân biệt 429 với lỗi khác.
- **Màn hội thoại** `speak-buddi/src/features/conversation/` (thay/ mở rộng `ConversationPlaceholder.jsx` — hiện chỉ là placeholder; story này thêm phần quota UI, phần hội thoại lõi do S7.1):

  - **Xử lý 429:** khi `/speak` hoặc heartbeat trả 429 → khóa input gửi tin, hiển thị **banner cảnh báo** đúng message SRS, kèm 2 CTA: "Nâng cấp Pro" (→ trang pricing/payment) và hiển thị thời điểm reset (`reset_at`) dạng đếm ngược/giờ cụ thể.
