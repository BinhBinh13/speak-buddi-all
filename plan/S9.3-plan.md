## 1. Mục tiêu & phạm vi

Triển khai **luồng crawl Langeek thành công (happy path)**: hệ thống chạy job định kỳ **hàng tuần**, lấy danh sách topic/từ vựng từ `https://langeek.co/en-VI/vocab/level-based`, **ánh xạ** sang bảng nội bộ `level` / `topic` / `topic_word` (A1–C2), rồi **tự động publish** (`source='langeek'`, `is_active=true`) **không cần Admin duyệt trước** (AC-13-04, BR10). Ghi **lịch sử crawl** (`content_crawl_job` + `content_crawl_log`) và cung cấp **Admin UI** theo mockup `langeek_crawler_admin` để xem trạng thái sync, log gần nhất, và **trigger thủ công** (dev/staging hoặc chạy lại ngoài lịch).

"Done" = job tuần (hoặc trigger thủ công qua API) crawl Langeek thật/giả lập → upsert topic/word vào DB → học viên thấy nội dung mới ở `/vocabulary` / roadmap; Admin thấy sync status + bảng log trên `/admin/crawler`; tiến độ học cũ được giữ (AC-05-04 / S3.4).

> **Phạm vi loại trừ rõ ràng (story khác):**
> - **Fallback khi Langeek lỗi** (giữ cache, retry, notify Admin) → **S9.4** (AC-13-05, BR12, FR-20). S9.3 chỉ implement hook/log cơ bản để S9.4 mở rộng; không bắt buộc email/notification đầy đủ.

> - **Tạo quiz tự động từ crawl** — không có trong AC-13-04; chỉ sync topic + word.
> - **Manual curation fallback khi crawl bị cấm pháp lý** — SRS TBD; S9.3 ghi guardrail stub + flag `crawler_enabled=false`.

### Backend (speak-buddi-be/)

#### 3.1 Schema mới — `db/schema_crawler.sql`

Bảng theo SRS §13.1 (chưa có trong repo — tạo mới, idempotent):

| Bảng | Mục đích |
|---|---|
| `content_source` | Cấu hình nguồn: `name`, `base_url` (default Langeek level-based URL), `is_enabled`, `schedule_cron` (default weekly), `rate_limit_ms`, `last_success_at`, `robots_checked_at`, `compliance_note` |
| `content_crawl_job` | Mỗi lần chạy: `id`, `source_id`, `trigger` (`scheduled` \| `manual`), `status` (`pending` \| `running` \| `success` \| `failed`), `started_at`, `finished_at`, `stats_json` (topics_upserted, words_upserted, words_disabled) |
| `content_crawl_log` | Chi tiết/log dòng: `job_id`, `level_code`, `topic_slug`, `severity` (`info` \| `warn` \| `error`), `message`, `payload_json` (optional, không log secret) |

Seed 1 row `content_source` mặc định trỏ `https://langeek.co/en-VI/vocab/level-based`.

#### 3.2 Repository — `repositories/crawl_repo.py`

- `get_active_source()`, `update_source_config(data)`
- `create_job(source_id, trigger)`, `finish_job(job_id, status, stats)`, `append_log(job_id, ...)`
- `list_jobs(limit, offset)`, `get_sync_status()` → `{ last_success_at, last_job_status, next_scheduled_at, cache_active: true }`

#### 3.3 Repository mở rộng — `repositories/content_repo.py`

Thêm hàm **upsert cho crawler** (tách khỏi Admin CRUD để không ghi đè `source='admin'`):

```python
async def upsert_langeek_topic(db, *, level_id, name, slug, description, display_order) -> dict
async def upsert_langeek_word(db, *, topic_id, level_id, word, phonetic, meaning_vi, meaning_en, example_sentence, display_order) -> dict
async def soft_disable_langeek_words_not_in_batch(db, topic_id, active_words: set[str]) -> int
```

Quy tắc upsert:
- Match topic theo **`slug`** (sinh từ Langeek slug/URL path, normalize lowercase-hyphen).
- Match word theo **`(topic_id, LOWER(word))`** — nếu tồn tại và `source='langeek'` → UPDATE fields + set `is_active=true`; nếu `source='admin'` → **không ghi đè** (log warn, skip).
- Từ có trong DB `source='langeek'` nhưng **không** có trong batch mới → `soft_delete_word` (is_active=false).

#### 3.5 Scraper integration — `speak-buddi-scrape/`

Implement (Node + Playwright):

| File | Mô tả |
|---|---|
| `src/langeek/crawl.js` | Entry: crawl all levels A1–C2 từ index page; follow topic links; extract word, ipa, meaning_vi/en, example |
| `src/langeek/parse.js` | Pure helpers + unit-testable selectors (TBD sau khi inspect DOM thật) |
| `src/cli.js` | `node src/cli.js --output batch.json` — stdout JSON cho BE subprocess |
| `package.json` | Scripts: `"crawl": "node src/cli.js"`, dependency `playwright` |

**Gọi từ BE:** `subprocess` chạy `node speak-buddi-scrape/src/cli.js` (path configurable `SCRAPE_ROOT`); timeout 10–15 phút; parse JSON stdout. Alternative: expose scraper qua HTTP localhost (chỉ dev) — ưu tiên subprocess đơn giản.

**Rate limit:** delay 500–1500ms giữa page requests (config `content_source.rate_limit_ms`).
