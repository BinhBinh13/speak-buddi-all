## 1. Mục tiêu & phạm vi

Hoàn thiện luồng **đăng xuất chủ động** cho cả khu học tập (Student/Paid) và khu quản trị (Admin):

1. Khi user/admin bấm "Đăng xuất" → xóa toàn bộ credential phía client (`token`, `refresh_token`, `user_info`).
2. Điều hướng về trang công khai (`/login` hoặc `/`) và chặn truy cập route bảo vệ.
3. Đồng bộ trạng thái auth giữa các tab/component (AuthContext, ProtectedRoute, AdminRoute).
4. Không log token/password (§4.5).

**Định nghĩa "done":** nút Đăng xuất hiển thị đúng vị trí trên sidebar user + admin; bấm logout → localStorage sạch → redirect → refresh/revisit protected route bị chặn; đa tab đồng bộ; không cần endpoint BE bắt buộc (JWT stateless v1.1).

**Ngoài scope (ghi TBD):**
- Revoke refresh token server-side / đăng xuất tất cả thiết bị (S8.x khi có `user_session` DB).
- `POST /api/auth/logout` — tùy chọn, chỉ thêm nếu muốn chuẩn bị revoke sau này; **không bắt buộc** cho story này.
- Google `google.accounts.id.disableAutoSelect()` — nice-to-have, không chặn done.

### 3.3 Frontend (`speak-buddi/`)

#### Task FE-1: Hook `useLogout` — một điểm vào duy nhất

Tạo `src/shared/auth/useLogout.js`:

```js
import { useCallback } from "react";
import { useNavigate } from "react-router-dom";
import { useAuth } from "./AuthContext";

/**
 * @param {{ redirectTo?: string }} [options] — mặc định "/login"
 */
export function useLogout({ redirectTo = "/login" } = {}) {
  const { logout } = useAuth();
  const navigate = useNavigate();

  return useCallback(() => {
    logout();
    // replace để không quay lại protected page bằng nút Back
    navigate(redirectTo, { replace: true });
  }, [logout, navigate, redirectTo]);
}
```

- Dùng `navigate` (React Router) thay vì `window.location` để giữ SPA, trừ khi cần hard reload (không cần cho story này).
- Caller đặc biệt (xóa tài khoản) có thể tiếp tục gọi `logout()` + `navigate("/")` trực tiếp.
