## 1. Mục tiêu

Story S4.5 hoàn chỉnh vòng đời làm bài kiểm tra từ phía user:

1. **QuizListPage** — màn `/quiz` liệt kê tất cả bài kiểm tra active, cho phép lọc theo **level** và/hoặc **topic**, hiển thị badge "Đã làm X lần" để user biết bài nào đã làm.
2. **Filter BE** — `GET /api/tests` mở rộng thêm param `level_id` (optional), hỗ trợ filter đơn lẻ hoặc kết hợp `topic_id + level_id`.
3. **Lịch sử attempt** — `GET /api/users/me/attempts` (hoặc `GET /api/tests/{test_id}/my-attempts`) để FE lấy số lần đã làm mỗi bài.
4. **Fix retry** — `QuizPage.handleRetry` gọi `startAttempt` tạo attempt mới thay vì chỉ reset state (tránh 409 khi nộp attempt cũ đã submitted).
5. **Snapshot invariant** — xác nhận bằng tài liệu: không cần thay đổi DB vì `quiz_attempt_answer.is_correct` đã là snapshot tại thời điểm submit, đáp ứng hoàn toàn AC-06-04.

**Kết quả:** Sau S4.5, EPIC-4 hoàn tất end-to-end: user chọn bài → làm → nộp → xem kết quả → làm lại (attempt mới). Nội dung mới (vocabulary cập nhật) không làm ảnh hưởng attempt cũ.

### 4.1 BE — Mở rộng filter tests

#### 4.1.1 Repo: `get_tests_by_filter()`

Thay thế chức năng của `get_tests_by_topic()` (vẫn giữ hàm cũ để không break caller cũ), thêm hàm mới hỗ trợ cả 2 params:

```python
async def get_tests_by_filter(
    db: AsyncSession,
    topic_id: str | None = None,
    level_id: str | None = None,
) -> list[dict]:
    """
    Lấy tất cả bài kiểm tra active, lọc theo topic và/hoặc level.
    Nếu cả 2 None → trả tất cả bài active (trang list không filter).
    S4.5 — AC-06-04: chỉ trả is_active=TRUE để dùng vocabulary mới nhất.
    """
    conditions = ["vt.is_active = TRUE"]
    params = {}

    if topic_id:
        conditions.append("vt.topic_id = CAST(:topic_id AS UUID)")
        params["topic_id"] = topic_id
    if level_id:
        conditions.append("vt.level_id = CAST(:level_id AS UUID)")
        params["level_id"] = level_id

    where_clause = " AND ".join(conditions)
    sql = f"""
        SELECT vt.id::text,
               vt.topic_id::text,
               vt.level_id::text,
               vt.title,
               vt.description,
               vt.is_active,
               vt.created_by::text,
               vt.created_at
        FROM   vocabulary_test vt
        WHERE  {where_clause}
        ORDER  BY vt.created_at ASC
    """
    r = await db.execute(text(sql), params)
    return [dict(row) for row in r.mappings().all()]
```

#### 4.1.2 Repo: `get_attempt_count_by_user()`

```python
async def get_attempt_count_by_user(
    db: AsyncSession,
    user_id: str,
    test_ids: list[str],
) -> dict[str, int]:
    """
    Đếm số lượt làm bài (status='submitted') của user cho từng test_id.
    Trả dict { test_id: count } — các test_id không có attempt trả 0.
    Chỉ đếm attempt đã submitted (không đếm in_progress bị bỏ dở).
    """
    if not test_ids:
        return {}
    r = await db.execute(
        text("""
            SELECT qa.vocabulary_test_id::text AS test_id,
                   COUNT(*)::int              AS cnt
            FROM   quiz_attempt qa
            WHERE  qa.user_id = CAST(:user_id AS UUID)
              AND  qa.vocabulary_test_id = ANY(CAST(:test_ids AS UUID[]))
              AND  qa.status = 'submitted'
            GROUP  BY qa.vocabulary_test_id
        """),
        {"user_id": user_id, "test_ids": test_ids},
    )
    result = {row["test_id"]: row["cnt"] for row in r.mappings().all()}
    # Điền 0 cho test chưa có attempt
    for tid in test_ids:
        result.setdefault(tid, 0)
    return result
```

#### 4.1.4 Endpoint: sửa `GET /api/tests`

```python
@router.get("/tests", response_model=list[VocabularyTestWithAttemptCountOut])
async def list_tests(
    topic_id: uuid_mod.UUID | None = None,
    level_id: uuid_mod.UUID | None = None,
    user: dict = Depends(current_user),
    db: AsyncSession = Depends(get_db),
) -> list[VocabularyTestWithAttemptCountOut]:
    """
    Trả danh sách bài kiểm tra active, lọc theo topic_id và/hoặc level_id.
    Kèm attempt_count: số lần user hiện tại đã làm mỗi bài (status='submitted').
    Nếu không truyền filter → trả tất cả bài active.
    """
    rows = await quiz_repo.get_tests_by_filter(
        db,
        topic_id=str(topic_id) if topic_id else None,
        level_id=str(level_id) if level_id else None,
    )
    if not rows:
        return []

    test_ids = [r["id"] for r in rows]
    count_map = await quiz_repo.get_attempt_count_by_user(db, user["sub"], test_ids)

    return [
        VocabularyTestWithAttemptCountOut(
            **row,
            attempt_count=count_map.get(row["id"], 0),
        )
        for row in rows
    ]
```

> **Backward-compat:** Caller cũ truyền `topic_id` vẫn hoạt động. `attempt_count` là field mới, không break consumer.

```js
const [retrying, setRetrying] = useState(false);

async function handleRetry() {
  setRetrying(true);
  setCurrentIndex(0);
  setUserAnswers({});
  setFinished(false);
  setSubmitError(null);
  setUnansweredIds(new Set());
  try {
    const attemptData = await startAttempt(testId);
    setAttemptId(attemptData?.id ?? null);
  } catch (err) {
    setError(err.message || "Không thể bắt đầu lượt làm mới. Vui lòng thử lại.");
  } finally {
    setRetrying(false);
  }
}
```

### 4.6 Snapshot invariant — AC-06-04

Đây là xác nhận kỹ thuật, **không cần thay đổi code**:

| Cơ chế | Triển khai |
|---|---|
| `quiz_attempt_answer.is_correct` | Ghi tại thời điểm submit (server-side), không tính lại sau |
| `upsert_attempt_answer()` | Chỉ UPDATE khi `quiz_attempt.status = 'in_progress'` (được guard bởi `submit_quiz()` → 409 nếu đã submitted) |

- **Không cần migration DB:** Schema đã đủ từ S4.1. Story này chỉ thêm query logic và UI.
- **`attempt_count` là soft feature:** Nếu endpoint lịch sử chưa ổn định, FE có thể bỏ badge và chỉ để "Chưa làm" — không block UI chính.
