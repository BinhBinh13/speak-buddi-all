- **Epic:** EPIC-6 — Luyện phát âm (UC08, F08)
- **Ưu tiên:** P2

- **Ngày lập:** 2026-06-07
- **Cập nhật:** 2026-06-07 — đơn giản hóa: **không lưu file audio**, chỉ lưu score + metadata

## 1. Quyết định thiết kế: không lưu file audio

SRS §4.6 ghi *"Audio pronunciation attempts are stored for 30 days"*, nhưng sau phân tích:

| Tiêu chí | Lưu audio file | Chỉ lưu score/metadata |
|---|---|---|
| UI có tính năng "nghe lại"? | Không (mockup không có) | ✅ đủ |
| Thuật toán chấm có re-score? | Web Speech API → kết quả ngay, không re-score | ✅ đủ |
| Độ phức tạp | Cao (filesystem, cleanup job 30 ngày, stream endpoint, security) | ✅ Thấp |
| Storage cost | Tiêu tốn disk | ✅ Không |
| Risk data leak | Restricted/Sensitive audio | ✅ Không có |

**Kết luận:** Với v1, lưu score + metadata là đủ. Nếu tương lai cần replay audio có thể bổ sung sau.

> ⚠️ **Backend-only story** — không có màn UI tương ứng.

## 2. Acceptance Criteria

- [ ] **AC-08-03 (storage):** Khi chấm điểm thành công, hệ thống lưu metadata/score vào DB; POST /api/pronunciation/attempts trả về attempt đã lưu.
- [ ] **§4.6 (metadata dài hạn):** Row `pronunciation_attempt` (word, score, feedback, created_at) không có TTL — lưu dài hạn.
- [ ] **§4.5 (no sensitive log):** Không log token/password/audio; chỉ log metadata an toàn (user_id, attempt_id, score).
- [ ] **§4.6-c:** Bảng này không liên quan AI conversation audio (endpoint chỉ phục vụ pronunciation).
- [ ] **Owner guard:** Chỉ user sở hữu mới đọc được lịch sử của mình.

## 3. Thiết kế kỹ thuật

### 3.1 Data model — bảng `pronunciation_attempt`

| Cột | Kiểu | Ghi chú |
|---|---|---|
| `id` | UUID PK `gen_random_uuid()` | |
| `user_id` | UUID NOT NULL FK `users(id)` ON DELETE CASCADE | |
| `word` | VARCHAR(255) NOT NULL | từ/cụm user luyện |
| `topic_word_id` | UUID NULL FK `topic_word(id)` ON DELETE SET NULL | link tùy chọn, null khi từ bị xóa |
| `score` | NUMERIC(5,2) NULL | điểm phát âm (0.00–100.00) |
| `feedback` | TEXT NULL | feedback text từ chấm điểm |
| `created_at` | TIMESTAMPTZ NOT NULL DEFAULT NOW() | |

- Index `idx_pa_user_created` ON `(user_id, created_at DESC)` — query lịch sử.
- KHÔNG có `audio_path`, `expires_at`, cleanup job.
- KHÔNG soft-delete (metadata giữ dài hạn).

### 3.2 API Endpoints

| Method | Path | Mô tả |
|---|---|---|
| `POST` | `/api/pronunciation/attempts` | Lưu 1 attempt; body JSON: `{word, topic_word_id?, score, feedback}` |
| `GET` | `/api/pronunciation/history` | Lịch sử attempt của user, phân trang `?limit=20&offset=0` |

**Auth:** Cả 2 endpoint yêu cầu JWT Bearer → `current_user`.
**POST response** (`PronunciationAttemptOut`): `{id, word, score, feedback, created_at}`.
**GET response**: `{items: [...], total, limit, offset}`.
