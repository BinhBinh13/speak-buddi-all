- **Epic:** EPIC-11 — Admin: Phân tích & Báo cáo (UC12, F14, FR-14, owner: Dev C)

## 1. Mục tiêu & phạm vi

Xây màn hình **Dashboard quản trị** (`/admin/dashboard`) cho Admin xem 4 nhóm chỉ số tổng quan: **doanh thu, người dùng, học tập (learning), và AI usage**, kèm các thẻ KPI và biểu đồ (recharts).

"Done" khi: Admin đăng nhập (role=admin) vào `/admin/dashboard` thấy các thẻ KPI (tổng user, free/paid, doanh thu, quiz attempt, tỉ lệ đúng/sai, thời lượng hội thoại AI) và biểu đồ user/doanh thu theo thời gian; khi có dữ liệu thì hiển thị số thật, khi chưa có thì hiển thị empty/zero state rõ ràng (không vỡ UI).

**Trong phạm vi S11.1:** API tổng quan (overview/summary), thẻ KPI, biểu đồ tổng quan, layout admin + route guard.
**Ngoài phạm vi (story khác):** Bộ lọc doanh thu chi tiết theo ngày/tháng/năm/gói = **S11.2**; Export Excel/PDF + lịch sử export = **S11.3**. Plan này phải đặt nền (API có thể tham số hóa thời gian) để S11.2/S11.3 mở rộng dễ dàng.

## 2. Acceptance Criteria cần thỏa

- [ ] **AC-12-01:** Admin mở analytics, khi có dữ liệu → hệ thống hiển thị các chỉ số **revenue, user, learning, AI usage**.
- [ ] **§3.6 (phủ phần overview của S11.1):** Dashboard hiển thị: tổng user, free vs paid, new users theo ngày/tháng, revenue tổng, quiz attempts, tỉ lệ đúng/sai, thời lượng hội thoại AI, từ vựng được học nhiều nhất.

## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)

Tạo module analytics theo đúng pattern hiện có (router + repo + schema, async SQLAlchemy, `Depends(require_admin)` từ `auth/deps.py`).

**File mới:**
- `routers/admin_analytics.py` — `APIRouter(prefix="/api/admin/analytics", tags=["admin-analytics"])`, mọi route `Depends(require_admin)`.
- `repositories/analytics_repo.py` — các hàm query thuần (SQL text qua AsyncSession, bám phong cách `quiz_repo`/`translate_repo`).
- `schemas/analytics.py` — Pydantic out-models cho response.

**Đăng ký router:** thêm `admin_analytics` vào import + `app.include_router(...)` trong `main.py`.

**Endpoint đề xuất (S11.1):**

`GET /api/admin/analytics/overview` → `DashboardOverviewOut`
Trả gói tổng quan đủ render dashboard 1 lần (giảm round-trip):

```jsonc
{
  "users": {
    "total": 0,
    "free": 0,
    "paid": 0,
    "new_today": 0,
    "new_this_month": 0
  },
  "revenue": {
    "total_vnd": 0,
    "this_month_vnd": 0,
    "currency": "VND",
    "is_estimated": true        // true vì chưa có bảng transaction thật (xem §6)
  },
  "learning": {
    "quiz_attempts_total": 0,
    "correct_answers": 0,
    "wrong_answers": 0,
    "accuracy_percent": 0.0,    // correct / (correct+wrong) * 100
    "avg_score_percent": 0.0,
    "top_words": [              // từ vựng được học nhiều nhất (most learned)
      { "word": "example", "learned_count": 0 }
    ]
  },
  "ai_usage": {
    "total_minutes": 0,
    "conversations": 0,
    "is_available": false       // false khi chưa có bảng AI usage (xem §6)
  }
}
```

`GET /api/admin/analytics/timeseries?metric={users|revenue}&range={7d|30d|year}` → `TimeseriesOut`
Dữ liệu cho biểu đồ (recharts). Mặc định `metric=users&range=7d`.

```jsonc
{
  "metric": "users",
  "range": "7d",
  "points": [ { "label": "2026-06-01", "value": 12 }, ... ]
}
```

> Thiết kế `range` param ngay từ S11.1 để S11.2 (lọc doanh thu theo ngày/tháng/năm/gói) chỉ cần mở rộng tham số `plan_id` + `granularity` thay vì đổi contract.

**Nguồn dữ liệu (mapping từng chỉ số → bảng có sẵn):**

| Chỉ số | Nguồn | Ghi chú |
|---|---|---|
| total users | `users` (status != 'deleted') | sẵn có |
| free vs paid | `users` LEFT JOIN `user_subscription` (status='active', expires_at > NOW()) | paid = có sub active; còn lại = free |
| new users by day/month | `users.created_at` group by ngày/tháng | sẵn có |
| revenue total/this_month | `user_subscription` (status='active') JOIN `payment_plan.price_vnd` | **ước lượng** — chưa có bảng giao dịch (§6) |
| quiz attempts | `quiz_attempt` (status='submitted') | sẵn có, có index `idx_qa_submitted` |
| correct/wrong, accuracy | SUM(`correct_answers`), SUM(`wrong_answers`) trên `quiz_attempt` | sẵn có |
| avg score | AVG(`score_percent`) | sẵn có |
| top learned words | `user_word_progress` JOIN `topic_word` group by word, ORDER BY count DESC LIMIT 10 | sẵn có (S3.3) |
| AI usage minutes/conversations | **CHƯA CÓ BẢNG** | trả `is_available=false` + 0; xem §6 |

### Data model / DB (nếu có)

- **Không tạo bảng mới cho revenue/user/learning** — tái dùng `users`, `user_subscription`, `payment_plan`, `quiz_attempt`, `user_word_progress`, `translation_history`.
- **AI usage:** chưa có bảng (Epic 7 chưa build). S11.1 **không** tạo bảng AI usage (thuộc trách nhiệm S7.x). Dashboard trả `ai_usage.is_available=false`. Khi S7.x thêm bảng (vd `ai_conversation` / `ai_quota_window`), bổ sung query vào `analytics_repo` sau (TBD §6).
- Index hiện có đủ: `idx_users_role_status`, `idx_user_subscription_user_status`, `idx_qa_submitted`. Cân nhắc thêm index trên `users.created_at` nếu timeseries new-users chậm (đánh giá khi đo P95).
