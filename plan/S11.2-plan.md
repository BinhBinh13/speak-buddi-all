## 1. Mục tiêu & phạm vi

Cho Admin **lọc và xem doanh thu** theo **khoảng thời gian** (ngày / tháng / năm / tổng) và **gói thanh toán** (`payment_plan`), hiển thị số liệu khớp bộ lọc trên dashboard admin.

"Done" khi: Admin trên `/admin/dashboard` chọn preset thời gian (hoặc from/to tùy chỉnh) + gói (Tất cả / một plan) → bấm Áp dụng → thấy **tổng doanh thu đã lọc** và **biểu đồ doanh thu** cập nhật theo filter; API trả dữ liệu từ **`payment_transaction` success** (không còn ước lượng subscription).

**Trong phạm vi S11.2:** endpoint revenue có filter, chuyển nguồn doanh thu sang giao dịch thật, UI filter trên dashboard, cập nhật chart khi `metric=revenue`.
**Ngoài phạm vi:** Export Excel/PDF + lịch sử export = **S11.3**; trang `/admin/reports` riêng (mockup `bao_cao_xuat_file_admin` — chỉ tham chiếu phần date/plan filter, không làm export engine).

## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)

Mở rộng module analytics S11.1 — **không tạo router mới**.

**Nguồn dữ liệu (thay đổi chính S11.2):**

| Trước (S11.1) | Sau (S11.2) |
|---|---|
| `user_subscription` × `payment_plan.price_vnd` | `payment_transaction` WHERE `status='success'` |
| `is_estimated=true` | `is_estimated=false` (khi có bảng; fallback 0 nếu chưa migrate schema_payment) |
| Thời gian: `starts_at` subscription | Thời gian: `paid_at` (index `idx_pt_status_paid`) |
| Số tiền: `price_vnd` plan | Số tiền: `amount_vnd` snapshot |

**Schema mới / sửa — `schemas/analytics.py`:**

```python
GranularityName = Literal["day", "month", "year", "total"]

class RevenueFilterOut(BaseModel):
    total_vnd: int
    transaction_count: int
    currency: str = "VND"
    is_estimated: bool = False
    granularity: GranularityName
    from_date: date | None = None   # inclusive
    to_date: date | None = None     # inclusive
    plan_id: UUID | None = None
    plan_name: str | None = None
    points: list[TimeseriesPoint] = []  # bucket theo granularity (total → 1 điểm hoặc [])
```

**Endpoint mới:**

`GET /api/admin/analytics/revenue`

| Query param | Mô tả |
|---|---|
| `granularity` | `day` \| `month` \| `year` \| `total` — preset §3.6 |
| `from` | ISO date `YYYY-MM-DD` (optional nếu dùng preset) |
| `to` | ISO date (optional) |
| `plan_id` | UUID gói (optional = tất cả) |

**Preset mapping (BE helper — tránh FE tính sai múi giờ):**

| `granularity` | Khoảng mặc định khi không truyền from/to |
|---|---|
| `day` | `paid_at` trong ngày hiện tại (server TZ UTC hoặc `date_trunc('day', NOW())`) |
| `month` | Tháng hiện tại |
| `year` | Năm hiện tại |
| `total` | Mọi `paid_at` NOT NULL (hoặc from/to rất rộng) |

Nếu **cả** `from` và `to` được truyền → override preset, vẫn group theo `granularity` (day→theo ngày, month→theo tháng, year→theo năm, total→1 tổng).

**Repo — `repositories/analytics_repo.py` (thêm/sửa):**

- `resolve_revenue_date_range(granularity, from_, to_) -> tuple[date, date]`
- `get_revenue_filtered(db, *, granularity, from_, to_, plan_id) -> dict` — 1–2 query:
  - SUM(`amount_vnd`), COUNT(*) với filter `status='success'`, `paid_at` trong range, optional `plan_id`
  - Series: `generate_series` + GROUP BY bucket (tái dùng pattern `get_timeseries`, nguồn `payment_transaction`)
- **Sửa** `get_revenue_stats()` — dùng cùng helper tháng hiện tại / tổng từ `payment_transaction` (overview dashboard đồng bộ S11.2)
- **Sửa** `get_timeseries(..., metric='revenue')` — gọi chung logic `payment_transaction` + hỗ trợ optional `plan_id` query param (mở rộng endpoint `/timeseries` hoặc delegate từ `/revenue`)

**Router — `routers/admin_analytics.py`:**

- Thêm `GET /revenue` → `RevenueFilterOut`, validate query (400 nếu from>to, plan_id không tồn tại → 404 hoặc 400)
- Mở rộng `GET /timeseries`: thêm optional `plan_id` (chỉ áp dụng khi `metric=revenue`)
- Log: `admin_id`, `granularity`, có/không `plan_id` — **không** log `total_vnd`

**Validation plan_id:** JOIN `payment_plan` hoặc lookup `payment_plan_admin_repo.get_by_id` — plan soft-deleted vẫn hiển thị trong filter admin (include inactive).
