## 1. Mục tiêu & phạm vi

Bổ sung hành động **Vô hiệu hóa (disable)** gói thanh toán trên màn Admin `/admin/payments`. Admin bấm Disable → **modal xác nhận** → BE set `payment_plan.is_active = FALSE` (AC-14-03). Gói bị disable **không còn hiển thị** cho học viên/guest ở pricing và **không thể checkout** mới; subscription đang active giữ nguyên (BR04 — trạng thái subscription độc lập).

"Done" = Admin disable được gói qua UI + API; sau disable, `GET /api/payment/plans` (public) và checkout không còn trả/chấp nhận gói đó; Admin vẫn thấy gói inactive (badge "Đã tắt" đã có từ S10.1) và có thể lọc trạng thái.

> **Phạm vi loại trừ:**
> - **Re-enable (`is_active=true`)** — AC-14-03 chỉ nói disable. `PUT` S10.1 đã nhận `is_active` nhưng **không** làm nút "Kích hoạt lại" trong S10.2 (có thể story sau nếu cần).
> - **Hard delete** — không có trong SRS §5.3.
> - **CRUD / validation** — S10.1 (không refactor trừ khi cần wire disable).

### Backend (speak-buddi-be/)

**Repository — bổ sung `repositories/payment_plan_admin_repo.py`:**
```python
async def soft_delete_plan(db: AsyncSession, plan_id: str) -> bool
```
- SQL: `UPDATE payment_plan SET is_active = FALSE, updated_at = NOW() WHERE id = :id`
- Trả `True` nếu `rowcount > 0` (idempotent — gọi lại trên row đã inactive vẫn match).
- **Không** `commit` trong repo (router commit — khớp `admin_content.py` DELETE pattern; khác `create_plan`/`update_plan` đã commit nội bộ).

**Router — mở rộng `routers/admin_payment_plan.py`** (giữ `dependencies=[Depends(require_admin)]`):

| Method | Path | Mô tả | Response |
|---|---|---|---|
| DELETE | `/api/admin/payment-plans/{plan_id}` | Soft-delete gói (`is_active=false`) | 204 / 404 |

- Log: `ADMIN_DISABLE_PAYMENT_PLAN  admin={sub}  plan={id}` (không log secret).
- 404 message: `"Gói thanh toán không tồn tại."` (tiếng Việt, khớp GET/PUT hiện có).

> **Không dùng** `PUT { is_active: false }` làm flow chính — DELETE semantic giống S9.2; `PUT is_active` giữ cho re-enable tương lai.

**Read-side (verify — không sửa nếu đã đúng):**
- `repositories/payment_repo.py` — `list_active_plans`, `get_active_plan` WHERE `is_active = TRUE`.
- Checkout `start_checkout` dùng `get_active_plan` → gói disabled → 404.
