## 1. Mục tiêu & phạm vi

Khi user click một node topic trên roadmap (`/roadmap`), mở `TopicModal` một màn hình duy nhất (không chia thành session card nhỏ) cho phép:
- Xem tổng quan topic: tên, level, tổng số từ, tiến độ học (từ `user_word_progress`).
- **"Add to Vocab"** — lưu topic vào danh sách học của user trong DB. `VocabularyPage` (Epic 3) sẽ hiển thị đúng những topic user đã add, không cần chọn level → topic nữa.
- Hai hành động chính: **"Học từ vựng"** (navigate thẳng vào topic) và **"Luyện với AI"** (navigate sang `/conversation` với batch từ được tính tự động).

**Session division — ẩn phía sau, chỉ dùng cho AI:**
Topic có thể có 50 từ; khi luyện với AI không muốn truyền 50 từ cùng lúc. Hệ thống tự chia thành các batch theo `user.words_per_session` (từ onboarding). User **không thấy UI chia batch**, nhưng khi bấm "Luyện với AI" app sẽ tự tìm batch đang dở (in_progress) hoặc bắt đầu batch 0, truyền đúng số từ đó sang conversation.

**Phạm vi KHÔNG bao gồm:**
- BatchAdjuster `[−] N [+]` — đã loại bỏ.
- Danh sách SessionCard trong modal — đã loại bỏ.
- Màn AI conversation thật (Epic 7) — chỉ định nghĩa data contract + route placeholder.
- Logic học từ vựng trong VocabularyPage (S3.2) — không sửa Flashcard/progress, chỉ đổi nguồn topic list.

## 3. Thiết kế kỹ thuật

### Backend

#### Migration mới — `db/migrations/006_user_session_progress.sql`

```sql
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- Tracking session AI conversation (batch progress)
CREATE TABLE IF NOT EXISTS user_session_progress (
    id           UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id      UUID        NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    topic_id     UUID        NOT NULL REFERENCES topic(id) ON DELETE CASCADE,
    batch_index  SMALLINT    NOT NULL,
    batch_size   SMALLINT    NOT NULL,
    status       VARCHAR(20) NOT NULL DEFAULT 'in_progress'
                             CHECK (status IN ('in_progress', 'completed')),
    started_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    completed_at TIMESTAMPTZ,
    UNIQUE (user_id, topic_id, batch_index)
);
CREATE INDEX IF NOT EXISTS idx_usp_user_topic
    ON user_session_progress (user_id, topic_id);

-- Topic user đã add vào danh sách học vocab
CREATE TABLE IF NOT EXISTS user_topic (
    user_id   UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    topic_id  UUID NOT NULL REFERENCES topic(id) ON DELETE CASCADE,
    added_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (user_id, topic_id)
);
CREATE INDEX IF NOT EXISTS idx_ut_user
    ON user_topic (user_id);
```

Đồng bộ vào `db/schema_progress.sql` (thêm block idempotent cuối file).

#### Endpoints — `routers/session.py` (MỚI, mount vào `main.py`)

Router prefix `/api`, tag `sessions`.

| Method | Path | Mô tả |
|---|---|---|
| `GET` | `/api/topics/{topic_id}/session-summary` | Trả `TopicSessionSummaryOut` — info để FE tính batch navigate AI. Guard `current_user`, 404 nếu inactive. |
| `PUT` | `/api/topics/{topic_id}/sessions/progress` | Upsert `user_session_progress`. Body `SessionProgressUpsert`. |
| `POST` | `/api/user/topics` | Add topic vào danh sách. Body `{ "topic_id": "..." }`. 409 nếu đã có (hoặc im lặng). |
| `DELETE` | `/api/user/topics/{topic_id}` | Remove topic khỏi danh sách. 204 No Content. |
| `GET` | `/api/user/topics` | Danh sách topic user đã add (`list[UserTopicOut]`). |

> Tái dùng `GET /api/topics/{topic_id}/words` (đã có ở `routers/learning.py` dòng 86) để lấy toàn bộ từ — FE slice client-side theo batch.

#### Navigate sang AI

```js
// Khi bấm "Luyện với AI":
const summary = await getTopicSessionSummary(node.id);
const batchSize = summary.batch_size;               // = words_per_session từ profile
const batchIndex = summary.resume_batch_index ?? 0; // batch in_progress hoặc 0
const allWords = await getTopicWords(node.id);
const batchWords = allWords.slice(
  batchIndex * batchSize,
  (batchIndex + 1) * batchSize
).map(w => ({ word: w.word, phonetic: w.phonetic, meaning_vi: w.meaning_vi,
               meaning_en: w.meaning_en, example_sentence: w.example_sentence }));

await startSession(node.id, { batchIndex, batchSize }); // mark in_progress
navigate("/conversation", {
  state: { topicId: node.id, topicName: node.name, batchIndex, words: batchWords },
});
```

## 4. Data flow

1. User click `RoadmapNode` → `setSelectedNode(node)`.
2. `TopicModal` mount: song song fetch `getTopicSessionSummary(id)` + `getUserTopics()` để biết đã add chưa.
3. Hiển thị: tổng quan + AddToVocabButton (toggle state) + ResumeBanner nếu `resume_batch_index != null`.
4. Bấm **"+ Thêm vào từ vựng"**: `addUserTopic(id)` → optimistic update → `VocabularyPage` sẽ hiện topic này lần sau.
5. Bấm **"Học từ vựng"**: navigate `/learn/:topicSlug` (VocabularyPage nhận `useParams` chọn thẳng topic).
6. Bấm **"Luyện với AI"**: tính batch từ summary → fetch words → startSession → navigate `/conversation` với state.
7. `VocabularyPage` load: `getUserTopics()` → hiện grid topic user đã add.
