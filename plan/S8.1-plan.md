## 1. Mục tiêu & phạm vi

Story này xây dựng **bước đầu của luồng thanh toán**: khi Student chọn một gói trả phí và bấm "Nâng cấp / Thanh toán", hệ thống **tạo một bản ghi `payment_transaction` ở trạng thái `pending`** và **trả về URL redirect tới payment provider** (hoặc khởi tạo provider flow), rồi chuyển hướng người dùng sang đó.

Vì **SRS để Payment Provider là TBD** (§2 Actor 5, §5.4, §5.5 Constraints, §6 Open Items — "Payment Provider specific service: TBD"), story này triển khai theo **kiến trúc provider-agnostic + một MockProvider mặc định** đủ để chạy end-to-end luồng UC10, đồng thời để chỗ cắm provider thật (VNPay/MoMo/PayOS/Stripe) sau này mà không phải sửa router/schema. Đây **không phải blocker** — xem mục 6.

**"Done" =**
- Có bảng `payment_transaction` (PostgreSQL) khớp §5.3 + draft schema.
- Endpoint `POST /api/payment/checkout` nhận `plan_id`, validate plan active, tạo transaction `pending`, trả `{ transaction_id, redirect_url }`.
- FE: nút "Nâng cấp Pro" (khi đã đăng nhập) gọi checkout → redirect sang `redirect_url`.
- MockProvider redirect tới một màn mock-pay nội bộ để S8.2 (webhook/callback) có thể tiếp nối.

**Ngoài phạm vi S8.1 (chuyển cho story sau):**
- Xử lý webhook/callback → kích hoạt subscription, set Paid User → **S8.2** (AC-10-02, BR04).
- Xử lý fail/cancel: giữ plan cũ + retry + email → **S8.3** (AC-10-03).
- Màn kết quả `thanh_toan_thanh_cong` / `thanh_toan_that_bai` → **S8.3** (mockup gắn ở đó).
- Đổi voice/model ElevenLabs → **S8.4**.
- CRUD payment_plan của Admin → **S10.1** (bảng `payment_plan` đã có sẵn từ S1.1).

## 3. Thiết kế kỹ thuật

### 3.0 Quyết định kiến trúc — Provider TBD

SRS chốt: provider TBD nhưng **chắc chắn dùng webhook/callback + redirect** (§5.4 "Webhook/callback + redirect"), và **không lưu card data, delegate cho provider** (§4.6 PCI-DSS). Vì vậy:

- Định nghĩa một interface provider thuần Python `PaymentProvider` với 2 method chính:
  - `create_checkout(transaction, plan, user) -> CheckoutResult(redirect_url, provider_ref)`
  - (dành cho S8.2) `verify_callback(payload) -> CallbackResult` — chưa implement đầy đủ ở S8.1, chỉ khai báo.
- Cài đặt **`MockPaymentProvider`** (mặc định, dùng được ngay): sinh `provider_ref` giả + `redirect_url` trỏ tới màn mock-pay nội bộ (`/payment/mock?tx=<id>`) để dev/QA bấm "Giả lập thành công/thất bại" → kích S8.2.
- Provider được chọn qua env `PAYMENT_PROVIDER` (default `mock`). Khi có provider thật, chỉ thêm 1 class mới + map trong factory, **không đụng router/schema/FE**.
- Lưu `provider` (string) + `provider_transaction_id` vào `payment_transaction` ngay từ S8.1 để khớp draft schema (`uq_payment_provider_tx`).

### 3.1 Data model / DB — `payment_transaction` (PostgreSQL)

Tạo file mới `speak-buddi-be/db/schema_payment.sql` (port draft MySQL → PostgreSQL, **khớp convention `schema_core.sql`**: UUID PK, TIMESTAMPTZ, trigger `set_updated_at`, idempotent `CREATE TABLE IF NOT EXISTS`).

> Lưu ý: `payment_plan` và `user_subscription` **đã có** trong `schema_core.sql` (S1.1). S8.1 chỉ thêm `payment_transaction`. Không sửa 2 bảng cũ.

| Cột | Kiểu | Ghi chú |
|---|---|---|
| id | UUID PK `gen_random_uuid()` | |
| user_id | UUID FK→users (CASCADE) NOT NULL | người mua |
| plan_id | UUID FK→payment_plan (RESTRICT) NOT NULL | gói được mua (tên cột khớp `user_subscription.plan_id`) |
| user_subscription_id | UUID FK→user_subscription (SET NULL) NULL | gắn khi S8.2 kích hoạt subscription |
| provider | VARCHAR(80) NOT NULL | 'mock' (default) / 'vnpay' / 'momo'… |
| provider_transaction_id | VARCHAR(255) NULL | mã từ provider; NULL lúc vừa tạo |
| amount_vnd | INTEGER NOT NULL CHECK (>= 0) | snapshot giá tại thời điểm mua (khớp `payment_plan.price_vnd` kiểu INTEGER) |
| currency | CHAR(3) NOT NULL DEFAULT 'VND' | |
| status | VARCHAR(20) NOT NULL DEFAULT 'pending' CHECK IN ('pending','success','failed','cancelled','refunded') | S8.1 chỉ tạo 'pending' |
| failure_reason | VARCHAR(500) NULL | điền ở S8.3 |
| paid_at | TIMESTAMPTZ NULL | set ở S8.2 |
| raw_payload | JSONB NULL | payload provider; **KHÔNG log** (§4.5) |
| created_at | TIMESTAMPTZ NOT NULL DEFAULT NOW() | |
| updated_at | TIMESTAMPTZ NOT NULL DEFAULT NOW() | trigger |

**Index / Constraint:**
- `uq_payment_provider_tx` UNIQUE `(provider, provider_transaction_id)` — chống xử lý webhook trùng (S8.2 cần). Lưu ý: nhiều row `NULL` provider_transaction_id được phép trong Postgres (NULL không vi phạm UNIQUE) → an toàn ở giai đoạn `pending`.
- `idx_pt_user` `(user_id)` — lịch sử giao dịch của user.
- `idx_pt_plan` `(plan_id)` — analytics doanh thu theo gói (S11.2).
- `idx_pt_status_paid` `(status, paid_at)` — lọc giao dịch thành công (dashboard S11.1).
- `idx_pt_subscription` `(user_subscription_id)`.

Trigger `updated_at`: thêm `'payment_transaction'` vào danh sách bảng trong block `DO $$` (đặt trong `schema_payment.sql`, dùng lại function `set_updated_at()` đã có).

**Quyết định amount:** dùng `amount_vnd INTEGER` (không DECIMAL như draft) để **đồng bộ với `payment_plan.price_vnd INTEGER`** trong `schema_core.sql` — giá VND không có phần thập phân.

### 3.2 Backend (`speak-buddi-be/`)

**Endpoint mới:** `POST /api/payment/checkout`

```
Auth:     Depends(current_user)  (JWT — pattern như routers/translate.py)
Request:  { "plan_id": "<uuid>" }
Response: { "transaction_id": "<uuid>", "redirect_url": "<url>", "provider": "mock", "amount_vnd": 99000 }
Errors:
  400  { "detail": "Plan này miễn phí, không cần thanh toán" }  — price_vnd == 0
  401  Unauthorized                                              — không có JWT
  404  { "detail": "Payment plan not found or inactive" }        — plan không tồn tại / is_active=false
  502  { "detail": "Payment provider error" }                    — provider init thất bại
```

(Tùy chọn) `GET /api/payment/plans` — trả list `payment_plan` active để FE pricing động (nếu muốn bỏ static PLANS ở FE). Có thể để S10.1, S8.1 vẫn dùng được vì FE truyền `plan_id` lấy từ list này; nếu chưa có endpoint plans thì S8.1 thêm luôn endpoint đọc plan tối thiểu để FE có `plan_id` thật.

**Files cần tạo:**

| File | Nội dung |
|------|----------|
| `db/schema_payment.sql` | DDL `payment_transaction` + index + trigger (idempotent) |
| `schemas/payment.py` | `CheckoutRequest(plan_id: UUID)`, `CheckoutResponse(transaction_id, redirect_url, provider, amount_vnd)`, (option) `PaymentPlanOut` |
| `repositories/payment_repo.py` | raw SQL async (pattern `translate_repo.py`): `get_active_plan(db, plan_id)`, `create_transaction(db, ...) -> dict`, `set_provider_ref(db, tx_id, provider_tx_id)`, `(option) list_active_plans(db)` |
| `services/payment_service.py` | logic: `start_checkout(db, user_id, plan_id) -> CheckoutResult`; gọi provider factory |
| `services/payment_providers/__init__.py` | factory `get_provider()` đọc env `PAYMENT_PROVIDER` |
| `services/payment_providers/base.py` | interface `PaymentProvider` + dataclass `CheckoutResult` |
| `services/payment_providers/mock_provider.py` | `MockPaymentProvider` — sinh provider_ref + redirect_url nội bộ |
| `routers/payment.py` | `POST /api/payment/checkout` (+ option GET plans) |

**Sửa:**
- `main.py`: `from routers import payment` + `app.include_router(payment.router)`.
- `.env.example` (nếu có): thêm `PAYMENT_PROVIDER=mock`, `FRONTEND_BASE_URL=http://localhost:5173`.
- `README.md`: thêm bước `psql -f db/schema_payment.sql` vào thứ tự chạy schema (sau `schema_core.sql`).

**Logic `start_checkout` (services/payment_service.py):**
1. `plan = await payment_repo.get_active_plan(db, plan_id)` → 404 nếu None.
2. Nếu `plan.price_vnd == 0` → 400 (gói free không thanh toán).
3. `tx = await payment_repo.create_transaction(db, user_id, plan_id, amount_vnd=plan.price_vnd, provider=provider.name, status='pending')`.
4. `result = provider.create_checkout(tx, plan, user)` → trả `redirect_url`, `provider_ref`.
5. `await payment_repo.set_provider_ref(db, tx.id, result.provider_ref)` (nếu provider trả ref ngay).
6. Log audit: `log.info("CHECKOUT user=%s plan=%s amount=%d tx=%s provider=%s", ...)` — **không log payload/token**.
7. Return `CheckoutResponse`.

**Convention:**
- Router: `prefix="/api"`, `tags=["payment"]`, logger `logging.getLogger("speakbuddi.payment")`.
- Repo: raw SQL `text()` + `AsyncSession` + `CAST(:x AS UUID)` (giống `translate_repo.py`).

### 3.4 Luồng redirect (UC10 sequence, phần S8.1)

```
Student (PricingPage)
   │ chọn gói Pro → bấm "Nâng cấp Pro"
   ▼
FE startCheckout(plan_id)
   │ POST /api/payment/checkout { plan_id }   (JWT)
   ▼
BE payment_service.start_checkout
   │ validate plan active + not free
   │ INSERT payment_transaction (status=pending)
   │ provider.create_checkout() → redirect_url + provider_ref
   │ UPDATE provider_transaction_id
   ▼
BE → { transaction_id, redirect_url }
   ▼
FE window.location.href = redirect_url
   ▼
[mock] /payment/mock?tx=...  → (S8.2 webhook / S8.3 fail) tiếp nối
[thật] trang provider (VNPay/MoMo…) → provider gọi webhook về BE (S8.2)
```
