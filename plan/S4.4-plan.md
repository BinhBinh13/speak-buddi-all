## 3. Kế hoạch BE

### 3.1 Repo mới: `get_attempt_result()`

Thêm hàm vào `quiz_repo.py` — join `quiz_attempt_answer` → `quiz_question` → `quiz_answer` để lấy context câu hỏi + đáp án đúng:

```sql
SELECT
    qaa.quiz_question_id::text,
    qq.question_text,
    qq.question_type,
    qaa.quiz_answer_id::text       AS user_answer_id,
    qaa.text_answer                AS user_text_answer,
    qaa.is_correct,
    -- Đáp án đúng (correct answer text, nullable cho flashcard/fill_blank)
    correct_ans.answer_text        AS correct_answer_text
FROM quiz_attempt_answer qaa
JOIN quiz_question qq ON qq.id = qaa.quiz_question_id
LEFT JOIN quiz_answer correct_ans
    ON correct_ans.quiz_question_id = qq.id
   AND correct_ans.is_correct = TRUE
WHERE qaa.quiz_attempt_id = CAST(:attempt_id AS UUID)
ORDER BY qaa.answered_at ASC NULLS LAST
```

> **Note fill_blank:** đáp án đúng lưu trong `quiz_answer` với `is_correct=TRUE` (đã seed ở S4.1). `correct_answer_text` trả về text gốc.  
> **Note flashcard:** không có `quiz_answer` rows → `correct_answer_text = NULL` (acceptable — flashcard self-rated).  
> **Note grammar_mapping:** nhiều `quiz_answer` là đúng → LEFT JOIN lấy `LIMIT 1` nếu cần, hoặc chỉ dùng `is_correct` snapshot từ `qaa`.

### 3.2 Schema mới

Thêm vào `schemas/quiz.py`:

```python
class QuizAnswerReview(BaseModel):
    """Chi tiết 1 câu trả lời trong review — trả về trang kết quả."""
    question_id: uuid.UUID
    question_text: str
    question_type: str
    user_answer_id: Optional[uuid.UUID]
    user_text_answer: Optional[str]
    correct_answer_text: Optional[str]
    is_correct: bool

class QuizAttemptResultOut(BaseModel):
    """Kết quả đầy đủ 1 lượt làm bài — dùng cho result page (S4.4)."""
    id: uuid.UUID
    vocabulary_test_id: uuid.UUID
    total_questions: int
    correct_answers: int
    wrong_answers: int
    score_percent: float
    status: str
    started_at: datetime
    submitted_at: Optional[datetime]
    duration_seconds: Optional[int]   # submitted_at - started_at
    answers: list[QuizAnswerReview]

    class Config:
        from_attributes = True
```

### 3.3 Endpoint mới

Thêm vào `routers/quiz.py`:

```
GET /api/attempts/{attempt_id}/result  →  QuizAttemptResultOut
```

Logic:
1. `get_attempt_by_id()` → 404 nếu không tồn tại.
2. Kiểm tra `attempt["user_id"] == user["sub"]` → 403 nếu không phải của user.
3. Nếu `status == "in_progress"` → trả 409 (chưa nộp, không có kết quả).
4. `get_attempt_result(attempt_id)` → lấy answers với question context.
5. Tính `duration_seconds = (submitted_at - started_at).total_seconds()`.
6. Trả `QuizAttemptResultOut`.

## 7. Rủi ro / Cảnh báo

| Rủi ro | Mức | Giải pháp |
|---|---|---|
| grammar_mapping có nhiều `quiz_answer` đúng → LEFT JOIN lấy nhiều row | Trung bình | JOIN kèm `LIMIT 1` hoặc dùng `DISTINCT ON (qaa.id)` |
| flashcard không có `quiz_answer` → `correct_answer_text = NULL` | Thấp | FE handle `null`, không hiển thị "đáp án đúng" cho flashcard |
| `duration_seconds` âm nếu clock skew | Thấp | Guard: `max(0, duration)` |
| User refresh result page trước khi submit xong | Thấp | 409 guard + thông báo "Bài chưa nộp" |
| Score circle CSS conic-gradient không support trên Safari cũ | Thấp | Dùng `background: conic-gradient(...)` — support Safari 12.1+ |
