# Plan — S12.3: Form liên hệ / hỗ trợ + support email

- **Epic:** EPIC-12 — Compliance & Support (SRS §4.6, §4.7 + extra-scope)
- **Ưu tiên:** P2
- **AC nguồn:** extra-scope (ngoài SRS v1.1.0) — tham chiếu `speak-buddi-docs/requirements-summary.md` §28 (*User support through contact form or support email*); bám NFR SRS §4.5 (security/log), §4.8/§4.9 (UI/accessibility), §5.2 (validation)
- **Ngày lập:** 2026-06-07

## 1. Mục tiêu & phạm vi

Cung cấp **kênh hỗ trợ công khai** để người dùng (Guest hoặc đã đăng nhập) gửi yêu cầu trợ giúp về **tài khoản, thanh toán, AI, kỹ thuật** qua form trên web; backend chuyển tiếp nội dung tới **email support** của đội vận hành (SMTP hiện có — cùng stack với reset password / payment failed).

"Done" = trang `/contact` (hoặc `/help`) hoạt động; footer "Liên hệ"/"Trợ giúp" và link hỗ trợ trên màn thanh toán thất bại trỏ tới trang form; submit thành công hiển thị thông báo tiếng Việt; email tới inbox support (hoặc skip an toàn + log warning nếu SMTP chưa cấu hình — pattern giống `send_reset_email`).

**Phạm vi:** FE (trang công khai + cập nhật link) + BE (endpoint public + `send_support_contact_email`). **Không** làm admin inbox / ticket DB / Resend SDK migration. **Không** bắt buộc đăng nhập. **Không** có mockup UI riêng — port shell từ `LegalLayout` / `PricingPage` + footer mockup.

**Ngoài phạm vi v1.1.0:** lưu ticket PostgreSQL, rate limit Redis, captcha, auto-reply email cho user, tích hợp Resend API thay SMTP.

## 2. Acceptance Criteria cần thỏa

- [ ] **AC-S12.3-1 (§28):** Có **form liên hệ công khai** (route `/contact`) với các trường tối thiểu: họ tên, email, chủ đề (select), nội dung tin nhắn.
- [ ] **AC-S12.3-2 (§28):** Submit form gọi BE; backend **gửi email** tới **`SMTP_USER`** (cùng hộp thư SMTP đã cấu hình cho reset password / payment failed) qua `email_service` — **không** thêm biến `SUPPORT_EMAIL` riêng.
- [ ] **AC-S12.3-3 (§5.2):** Validation server-side: email hợp lệ (`validate_email`), họ tên/message không rỗng sau trim, độ dài giới hạn (name ≤100, subject ≤120, message ≤2000); lỗi trả message tiếng Việt.
- [ ] **AC-S12.3-4 (§4.8/§4.9):** UI tiếng Việt, responsive (375/768/1024), nút submit touch target ≥44px, lỗi hiển thị bằng text (`role="alert"`), bám design tokens `DESIGN.md` (primary `#3525cd`, font Be Vietnam Pro).
- [ ] **AC-S12.3-5 (§4.5):** Không log email/nội dung message trong application log; chỉ log event `SUPPORT_CONTACT ok` hoặc `skip`/`error` (type exception).
- [ ] **AC-S12.3-6:** Cập nhật **PublicFooter**: "Liên hệ" và "Trợ giúp" → `<Link to="/contact">` (thay `href="#"`).
- [ ] **AC-S12.3-7 (khuyến nghị):** Cập nhật **PrivacyPolicyPage** mục 8 — thêm link tới `/contact` bên cạnh `mailto:`; cập nhật **PaymentResultPage** link "Cần hỗ trợ?" → `/contact` (mockup `thanh_toan_that_bai_desktop`).
- [ ] **AC-S12.3-8 (UX):** Nếu user **đã đăng nhập**, pre-fill họ tên + email từ `AuthContext` (read-only email hoặc editable — khuyến nghị editable với default từ session).
- [ ] **AC-S12.3-9:** Trạng thái success sau submit: ẩn form, hiển thị hộp xác nhận ("Chúng tôi đã nhận yêu cầu… phản hồi trong 24–48 giờ") — pattern giống `ForgotPasswordPage` success state.

## 3. Thiết kế kỹ thuật

### Backend (speak-buddi-be/)

**Endpoint mới** — `routers/support.py` (router mới):

```
POST /api/support/contact
Auth: không bắt buộc (public)
Body: {
  "name": "Nguyen Van A",
  "email": "user@example.com",
  "subject": "payment",        // enum hoặc string
  "message": "Nội dung..."
}
Response 200: { "message": "Yêu cầu của bạn đã được gửi. Chúng tôi sẽ phản hồi qua email." }
Errors 400: validation tiếng Việt
Errors 502/503 (tùy chọn): SMTP lỗi nặng — hoặc vẫn 200 + log error nếu giữ pattern "skip an toàn" (khuyến nghị: 200 khi đã validate + đã cố gửi; 503 chỉ khi SMTP configured nhưng send fail — để user biết thử lại)
```

**Schema** — `schemas/support.py`:
- `ContactRequest`: `name: str`, `email: str`, `subject: str`, `message: str` (Pydantic `Field` max_length)
- `ContactResponse`: `message: str`
- `SUBJECT_CHOICES` constant: `account`, `payment`, `ai`, `technical`, `other` → label tiếng Việt map trong router hoặc FE

**Email** — mở rộng `services/email_service.py`:

```python
def send_support_contact_email(
    *,
    sender_name: str,
    sender_email: str,
    subject_key: str,
    message: str,
    user_id: str | None = None,
) -> None
```

- **Đích nhận:** `SMTP_USER` (cùng mailbox `.env` — vd `ngaidang53@gmail.com`). `From` = `SMTP_USER`; `To` = `SMTP_USER`; `Reply-To` = email user gửi form → support trả lời trực tiếp cho user.
- HTML email inline (Be Vietnam Pro, primary `#3525cd`) — body chứa: tên, email reply-to, chủ đề, message, user_id (nếu có), timestamp UTC.
- Skip + `log.warning` nếu `SMTP_USER/SMTP_PASS` trống (giống reset email) — dev env skip → 200 với message chung; production phải cấu hình SMTP.

**Config:** **Không thêm** `SUPPORT_EMAIL`. Tái dùng `SMTP_USER` / `SMTP_PASS` / `SMTP_HOST` / `SMTP_PORT` trong `core/config.py` (đã có).

**Đăng ký router** — `main.py`: `from routers import support` + `app.include_router(support.router)`.

**Optional auth enrichment:** dùng `Depends(get_optional_user)` nếu tạo helper mới trong `auth/deps.py` (try JWT, không fail nếu thiếu) — gắn `user_id` vào email footer. **Không bắt buộc** cho PASS; có thể hardcode `user_id=None` v1.

**Honeypot chống bot đơn giản (khuyến nghị):** field ẩn `website` — nếu có giá trị → 200 noop, không gửi mail.

### Frontend (speak-buddi/)

**Route** — `App.jsx`:
```jsx
<Route path="/contact" element={<ContactPage />} />
```

**Trang** — `src/features/support/ContactPage.jsx`:
- Shell: `PublicNavbar` + card form căn giữa (max-width ~560px) + `PublicFooter` — copy structure từ `PricingPage` / `LegalLayout` (padding, background `UI.background`).
- Form fields:
  - Họ tên (text, required)
  - Email (email, required)
  - Chủ đề (select): Tài khoản / Thanh toán & gói Pro / Hội thoại AI / Lỗi kỹ thuật / Khác
  - Nội dung (textarea, required, rows 5)
  - Honeypot `website` (hidden, tabIndex -1)
- Submit → `supportService.submitContact(payload)`
- Loading state trên nút; errors inline + `role="alert"`
- Success state (AC-S12.3-9)

**Service** — `src/features/support/services/supportService.js`:
```javascript
export async function submitContact(body) {
  return apiClient.post("/api/support/contact", body);
}
```
Dùng shared `apiClient` (cùng pattern `profileService.js` / `authService.js`).

**Cập nhật link:**
