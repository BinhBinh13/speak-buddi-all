## 3. Thiết kế kỹ thuật

### Approach chấm phát âm (quyết định kỹ thuật)

Hệ thống **không có Whisper/Azure Speech pre-tích hợp**; stack hiện có là **Anthropic (claude-haiku-4-5)** + **ElevenLabs TTS** + **Web Speech API (SpeechRecognition)** ở FE. So sánh phương án:

| Phương án | Ưu | Nhược | Kết luận |
|---|---|---|---|
| **A. SpeechRecognition (FE) lấy transcript → Anthropic (BE) chấm so target** | Tái dùng đúng hạ tầng đã có (`speechService.createSpeechRecognizer`, `get_claude_client`); không thêm dependency/chi phí mới; trả được feedback tiếng Việt + breakdown âm tiết dạng JSON | Không phải acoustic scoring thật (chấm dựa trên text user nói được nhận diện, không phải sóng âm) → độ chính xác phụ thuộc SpeechRecognition | **CHỌN cho v1.1.0** — đáp ứng AC-08-03 (có score/feedback + lưu metadata), chi phí thấp, đúng định hướng "có thể dùng Anthropic" trong đề bài |
| B. Azure Pronunciation Assessment / external speech API | Acoustic scoring chuẩn (phoneme-level) | Thêm vendor + key + chi phí + bảo mật audio; ngoài phạm vi hạ tầng hiện tại | Để **TBD** cho phiên bản sau, ghi ở §6 |
| C. Whisper self-host | Không phụ thuộc vendor | Hạ tầng GPU/độ trễ; quá nặng cho scope | Loại |

**Chốt phương án A.** Luồng:
1. FE (S6.1 đã có `audioBlob`; S6.2 bổ sung lấy **transcript**): dùng lại `createSpeechRecognizer` để nhận diện text user đọc trong lúc ghi (chạy song song MediaRecorder), HOẶC chạy SpeechRecognition là nguồn transcript chính. Đơn giản nhất: **dùng SpeechRecognition lấy transcript**, MediaRecorder blob để dành S6.3 lưu audio.
2. FE gọi `POST /api/pronunciation/score` với `{ target_text, transcript }`.
3. BE gửi Anthropic prompt: so `transcript` với `target_text`, trả **JSON** `{ overall, accuracy, fluency, syllables: [{text, score}], feedback }`.
4. BE lưu `pronunciation_attempt` + `pronunciation_syllable_score`, trả response cho FE.
5. FE render Score Display.

> Lưu ý: prompt yêu cầu Anthropic trả **JSON thuần** (no markdown); parse an toàn (try/except → nếu parse fail, raise 502 với message §5.2 Anthropic).

### Backend (speak-buddi-be/)

**Schema Pydantic** — `speak-buddi-be/schemas/pronunciation.py` (mới):
```python
class SyllableScore(BaseModel):
    text: str
    score: float          # 0..100

class ScoreRequest(BaseModel):
    target_text: str
    transcript:  str
    topic_word_id: str | None = None   # NULL khi dùng từ hardcode

class ScoreResponse(BaseModel):
    attempt_id: str
    overall:    float
    accuracy:   float
    fluency:    float
    syllables:  list[SyllableScore]
    feedback:   str       # tiếng Việt
```

**Service chấm điểm** — `speak-buddi-be/services/pronunciation_service.py` (mới):
- `score_pronunciation(target_text, transcript) -> dict` — build system prompt (mẫu giống `ai_service._build_system_prompt`), gọi `get_claude_client().messages.create(model="claude-haiku-4-5-20251001", max_tokens=300, ...)`, parse JSON, clamp các điểm về [0,100].
- Prompt yêu cầu: chấm `accuracy` (transcript khớp target tới đâu), `fluency`, `overall`, tách `target_text` thành âm tiết kèm điểm, viết `feedback` tiếng Việt ≤ 2 câu, **trả JSON only**.

**Repository** — `speak-buddi-be/repositories/pronunciation_repo.py` (mới), pattern raw SQL `text()` async giống `session_repo.py`:
- `insert_attempt(db, user_id, target_text, overall, accuracy, fluency, feedback, topic_word_id) -> attempt_id` (INSERT … RETURNING id).
- `insert_syllable_scores(db, attempt_id, syllables: list[dict])` (bulk insert theo `display_order`).
- 1 transaction: insert attempt → insert syllables → `db.commit()`.

**Router** — `speak-buddi-be/routers/pronunciation.py` (mới):
```python
router = APIRouter(prefix="/api/pronunciation", tags=["pronunciation"])

@router.post("/score", response_model=ScoreResponse)
async def score(body: ScoreRequest, payload=Depends(current_user), db=Depends(get_db)):
    if not body.transcript.strip():
        raise HTTPException(400, detail="No audio / empty transcript")   # FE map → §5.2 No audio
    try:
        result = score_pronunciation(body.target_text, body.transcript)
    except Exception:
        raise HTTPException(502, detail="AI scoring error")              # FE map → §5.2 Anthropic
    attempt_id = await pronunciation_repo.insert_attempt(db, payload["sub"], body.target_text, ...)
    await pronunciation_repo.insert_syllable_scores(db, attempt_id, result["syllables"])
    return ScoreResponse(attempt_id=attempt_id, **result)
```
- Guard JWT bằng `current_user` (giống `session.py`).

**Đăng ký router** — `speak-buddi-be/main.py`: thêm `pronunciation` vào import + `app.include_router(pronunciation.router)`.

### Data model / DB

Dùng đúng entity SRS §5.3. **schema-draft.sql là bản MySQL cũ** — repo thật dùng **PostgreSQL + UUID PK + migration idempotent** (xem `db/migrations/006_*.sql`). Tạo migration mới:

`speak-buddi-be/db/migrations/007_pronunciation.sql` (mới):
```sql
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

CREATE TABLE IF NOT EXISTS pronunciation_attempt (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    topic_word_id UUID REFERENCES topic_word(id) ON DELETE SET NULL,
    target_text   VARCHAR(1000) NOT NULL,
    audio_url     VARCHAR(500),                 -- NULL ở S6.2, S6.3 sẽ dùng
    overall_score  NUMERIC(5,2) CHECK (overall_score  BETWEEN 0 AND 100),
    accuracy_score NUMERIC(5,2) CHECK (accuracy_score BETWEEN 0 AND 100),
    fluency_score  NUMERIC(5,2) CHECK (fluency_score  BETWEEN 0 AND 100),
    feedback      TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_pron_attempt_user ON pronunciation_attempt (user_id, created_at);

CREATE TABLE IF NOT EXISTS pronunciation_syllable_score (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    attempt_id    UUID NOT NULL REFERENCES pronunciation_attempt(id) ON DELETE CASCADE,
    syllable_text VARCHAR(100) NOT NULL,
    score         NUMERIC(5,2) NOT NULL CHECK (score BETWEEN 0 AND 100),
    display_order SMALLINT NOT NULL DEFAULT 0
);
CREATE INDEX IF NOT EXISTS idx_pron_syl_attempt ON pronunciation_syllable_score (attempt_id);
```
- Bỏ `speaking_session_id` FK ở S6.2 (chưa có bảng `speaking_session` trong DB thật) — nếu cần liên kết session để dành story Epic 7; ghi TBD.
- Bỏ `is_active`/soft delete: SRS §5.3 chỉ yêu cầu soft delete cho `topic/topic_word/vocabulary_test/payment_plan`, không cho pronunciation.
