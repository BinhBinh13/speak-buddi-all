## 1. Mục tiêu & phạm vi

Tạo nhóm bảng **Quiz/Test** trong DB (PostgreSQL) làm nền cho toàn bộ tính năng bài kiểm tra từ vựng theo SRS §5.3:

- `vocabulary_test` — bài kiểm tra (tiêu đề, gắn topic/level, soft-delete)
- `quiz_question` — câu hỏi với 4 loại: `flashcard`, `multiple_choice`, `fill_blank`, `grammar_mapping`
- `quiz_answer` — đáp án cho từng câu hỏi (trắc nghiệm/mapping), với cờ `is_correct`
- `quiz_attempt` — lượt làm bài của user (trạng thái, điểm, timestamps)
- `quiz_attempt_answer` — câu trả lời cụ thể của user trong từng lượt làm

Story này cũng tạo:
- File SQL DDL idempotent: `speak-buddi-be/db/schema_quiz.sql`
- Pydantic schemas skeleton: `speak-buddi-be/schemas/quiz.py`
- Repository layer (raw SQL): `speak-buddi-be/repositories/quiz_repo.py`
- Service layer: `speak-buddi-be/services/quiz_service.py`
- Seed data mẫu: 1 vocabulary_test + 5 quiz_question đa dạng loại + quiz_answer

**"Done" =** file SQL chạy được trên PostgreSQL sau `schema_learning.sql`, tạo đủ 5 bảng + index + trigger + seed mẫu; Pydantic schemas đúng kiểu; repository CRUD cơ bản; service layer đơn giản. **Story này KHÔNG viết route/endpoint API** (đó là S4.2/S4.3/S4.4).

## 3. Thiết kế kỹ thuật

### 3.1 ERD tóm tắt

```
level (1) ──< (N) vocabulary_test ──< (N) quiz_question ──< (N) quiz_answer
                        │                      │
                 topic (N)              topic_word (N, nullable)
                        │
               quiz_attempt >──< (N) quiz_attempt_answer
                        │
                users (N)
```

Chi tiết quan hệ:
- `vocabulary_test.topic_id` → `topic.id` (SET NULL on delete)
- `vocabulary_test.level_id` → `level.id` (SET NULL on delete)
- `vocabulary_test.created_by` → `users.id` (SET NULL on delete)
- `quiz_question.vocabulary_test_id` → `vocabulary_test.id` (CASCADE)
- `quiz_question.topic_word_id` → `topic_word.id` (SET NULL) — nullable, flashcard có thể không gắn từ cụ thể
- `quiz_answer.quiz_question_id` → `quiz_question.id` (CASCADE)
- `quiz_attempt.user_id` → `users.id` (CASCADE)
- `quiz_attempt.vocabulary_test_id` → `vocabulary_test.id` (CASCADE)
- `quiz_attempt_answer.quiz_attempt_id` → `quiz_attempt.id` (CASCADE)
- `quiz_attempt_answer.quiz_question_id` → `quiz_question.id` (CASCADE)
- `quiz_attempt_answer.quiz_answer_id` → `quiz_answer.id` (SET NULL) — nullable cho fill_blank/flashcard

- `grammar_mapping` — nối câu/từ, cấu trúc tương tự multiple_choice nhưng FE sắp xếp

### 3.3 Pydantic Schemas (`schemas/quiz.py`)

```python
# Đề xuất cấu trúc schemas/quiz.py — skeleton, chưa wire vào app

# ── VocabularyTest ─────────────────────────────────────────────────────────────
class VocabularyTestCreate(BaseModel):
    topic_id: Optional[uuid.UUID]
    level_id: Optional[uuid.UUID]
    title: str = Field(..., max_length=200)
    description: Optional[str]

class VocabularyTestOut(VocabularyTestCreate):
    id: uuid.UUID
    is_active: bool
    created_by: Optional[uuid.UUID]
    created_at: datetime

# ── QuizAnswer ─────────────────────────────────────────────────────────────────
class QuizAnswerCreate(BaseModel):
    answer_text: str
    is_correct: bool = False
    display_order: int = 0

class QuizAnswerOut(QuizAnswerCreate):
    id: uuid.UUID

# ── QuizQuestion ───────────────────────────────────────────────────────────────
QuestionType = Literal["flashcard", "multiple_choice", "fill_blank", "grammar_mapping"]

class QuizQuestionCreate(BaseModel):
    vocabulary_test_id: uuid.UUID
    topic_word_id: Optional[uuid.UUID]
    question_text: str
    question_type: QuestionType
    display_order: int = 0
    answers: list[QuizAnswerCreate] = []

class QuizQuestionOut(BaseModel):
    id: uuid.UUID
    vocabulary_test_id: uuid.UUID
    topic_word_id: Optional[uuid.UUID]
    question_text: str
    question_type: str
    display_order: int
    answers: list[QuizAnswerOut]

# ── QuizAttempt ────────────────────────────────────────────────────────────────
class QuizAttemptOut(BaseModel):
    id: uuid.UUID
    user_id: uuid.UUID
    vocabulary_test_id: uuid.UUID
    total_questions: int
    correct_answers: int
    wrong_answers: int
    score_percent: float
    status: str
    started_at: datetime
    submitted_at: Optional[datetime]

# ── QuizAttemptAnswer ──────────────────────────────────────────────────────────
class QuizAttemptAnswerSubmit(BaseModel):
    quiz_question_id: uuid.UUID
    quiz_answer_id: Optional[uuid.UUID]  # None cho fill_blank/flashcard
    text_answer: Optional[str]           # fill_blank

class QuizAttemptAnswerOut(QuizAttemptAnswerSubmit):
    id: uuid.UUID
    quiz_attempt_id: uuid.UUID
    is_correct: bool
    answered_at: Optional[datetime]
```

## 5. Test & cách verify

```sql
-- 1. Kiểm tra 5 bảng tồn tại
\dt vocabulary_test quiz_question quiz_answer quiz_attempt quiz_attempt_answer

-- 2. Kiểm tra constraint question_type
INSERT INTO quiz_question (vocabulary_test_id, question_text, question_type)
VALUES (..., 'test', 'invalid_type');
-- Kỳ vọng: ERROR — vi phạm CHECK constraint

-- 3. Kiểm tra soft-delete vocabulary_test
UPDATE vocabulary_test SET is_active = FALSE WHERE title LIKE 'Test:%';
SELECT count(*) FROM vocabulary_test;                         -- row vẫn còn
SELECT count(*) FROM vocabulary_test WHERE is_active = TRUE;  -- giảm đi

-- 4. Kiểm tra UNIQUE (quiz_attempt_id, quiz_question_id) trong quiz_attempt_answer
-- Insert 2 lần cùng attempt_id + question_id → ERROR unique constraint

-- 5. Kiểm tra nullable quiz_answer_id (flashcard/fill_blank)
INSERT INTO quiz_attempt_answer (quiz_attempt_id, quiz_question_id, quiz_answer_id, is_correct)
VALUES (:attempt_id, :question_id, NULL, TRUE);
-- Kỳ vọng: SUCCESS (NULL được phép)

-- 6. Kiểm tra seed data
SELECT vt.title, COUNT(qq.id) AS question_count
FROM vocabulary_test vt
LEFT JOIN quiz_question qq ON qq.vocabulary_test_id = vt.id
WHERE vt.title LIKE 'Test: Greetings%'
GROUP BY vt.title;
-- Kỳ vọng: 1 row với question_count = 5

-- 7. Kiểm tra đa dạng question_type trong seed
SELECT question_type, COUNT(*) FROM quiz_question
WHERE vocabulary_test_id = (
    SELECT id FROM vocabulary_test WHERE title LIKE 'Test: Greetings%'
)
GROUP BY question_type;
-- Kỳ vọng: flashcard=1, multiple_choice=2, fill_blank=1, grammar_mapping=1

-- 8. Kiểm tra trigger updated_at
UPDATE vocabulary_test SET description = 'updated' WHERE title LIKE 'Test:%';
SELECT title, updated_at FROM vocabulary_test WHERE title LIKE 'Test:%';
-- updated_at phải đổi sang thời điểm hiện tại

-- 9. Idempotent — chạy lại lần 2 không lỗi
-- psql -U postgres -d speakbuddi -f db/schema_quiz.sql → pass
```

### Rủi ro

1. **`question_type` enum vs CHECK constraint:** Draft SRS dùng MySQL ENUM, PostgreSQL dùng CHECK constraint string — đã chọn `VARCHAR(30) CHECK IN (...)` để khớp pattern codebase (`user.role`, `user.status` trong `schema_core.sql`). Nếu sau này cần add type mới (vd `listen_choose` như draft SQL), chỉ cần `ALTER TABLE quiz_question DROP CONSTRAINT ... ADD CONSTRAINT ...` — không migration phức tạp.

2. **`quiz_answer` không có cho `flashcard`/`fill_blank`:** `quiz_attempt_answer.quiz_answer_id` phải nullable. Service layer cần xử lý đúng case NULL khi chấm điểm — flashcard do user tự đánh giá (is_correct do client gửi), fill_blank so sánh text. Ghi chú này cần nhắc developer S4.3/S4.4.

3. **Draft SRS dùng MySQL schema (BIGINT, ENUM, ENGINE=InnoDB):** KHÔNG copy nguyên văn. File `speak-buddi-docs/db/schema-draft.sql` dùng MySQL syntax. Toàn bộ DDL phải dùng PostgreSQL (UUID PK, TIMESTAMPTZ, VARCHAR CHECK) để khớp `schema_core.sql` và `schema_learning.sql`.

4. **`quiz_attempt_answer.is_correct` snapshot:** Khi nội dung test được admin sửa sau khi user đã làm bài (AC-06-04), `is_correct` trong `quiz_attempt_answer` KHÔNG được tính lại — đây là snapshot lịch sử. Repository `submit_attempt()` phải set `is_correct` tại thời điểm nộp, không tham chiếu `quiz_answer.is_correct` sau này.

5. **Seed data phụ thuộc seed S3.1:** Seed quiz test gắn với topic "greetings-introductions" (đã có trong `schema_learning.sql`). Nếu seed S3.1 chưa chạy, foreign key sẽ fail. → Bắt buộc chạy `schema_learning.sql` trước `schema_quiz.sql` (đã ghi trong thứ tự).

6. **Performance với nhiều attempt:** `quiz_attempt` không có giới hạn số lượt làm — user có thể làm lại nhiều lần. Index `(user_id, vocabulary_test_id)` đủ cho S4.5 history. Pagination sẽ cần ở S4.5 nếu history nhiều — ghi nhận nhưng chưa xử lý trong S4.1.
