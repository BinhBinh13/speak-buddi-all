## 1. Mục tiêu & phạm vi

Cho phép **Paid User** mở màn "Cài đặt giọng đọc", xem danh sách voice/model ElevenLabs đang active, chọn 1 voice và **lưu preference** để các luồng đọc audio (TTS `/tts`, hội thoại `/speak`) dùng voice đó thay cho voice mặc định. **Free Student** bị **chặn** (403 ở BE + redirect/overlay mời nâng cấp ở FE) theo BR06.

"Done" = Paid User chọn & lưu được voice (persist DB), reload vẫn giữ lựa chọn; free user không vào/không lưu được, được mời nâng cấp; `/tts` & `/speak` đọc preference của user khi có.

**Trong phạm vi:** danh sách voice (GET), preference của user (GET + PUT), guard paid-user, trang FE `/settings/voice` theo mockup, wiring TTS dùng preference.
**Ngoài phạm vi (ghi rõ ở mục 6):** Admin CRUD bảng voice (chỉ seed cứng/seed SQL ở story này), tính năng "Speaking Rate" và "Auto-Play" trong mockup (không nằm trong AC — đề xuất ẩn hoặc đánh dấu TODO story sau), preview audio voice (tùy chọn, không bắt buộc theo AC).

## 3. Thiết kế kỹ thuật

### Data model / DB
Theo §5.3 SRS, 2 entity cần có: `elevenlabs_voice_model` (danh mục voice) + `user_voice_preference` (lựa chọn của user). Hiện chưa có bảng nào (các schema `schema_core/learning/progress/quiz/payment.sql` không chứa).

- **File mới:** `speak-buddi-be/db/schema_voice.sql` (idempotent, theo pattern `schema_payment.sql`; chạy sau `schema_core.sql` vì FK tới `users`).
  - **`elevenlabs_voice_model`**
    - `id UUID PK DEFAULT gen_random_uuid()`
    - `voice_id TEXT NOT NULL UNIQUE` — ElevenLabs voice id (vd `pNInz6obpgDQGcFmaJgB`)
    - `model_id TEXT NOT NULL DEFAULT 'eleven_multilingual_v2'`
    - `display_name TEXT NOT NULL` (vd "Sarah"), `accent/language TEXT` (vd "US English"), `gender TEXT` (cho icon face_3/face), `is_pro BOOLEAN DEFAULT false` (badge PRO trong mockup — informational)
    - `is_active BOOLEAN NOT NULL DEFAULT true` (soft-disable — §5.3 policy active/soft-delete)
    - `sort_order INT DEFAULT 0`, `created_at/updated_at` + trigger `set_updated_at` (đã có trong `schema_core.sql`)
  - **`user_voice_preference`**
    - `user_id UUID PK REFERENCES users(id) ON DELETE CASCADE` (1-1, mỗi user 1 preference)
    - `voice_model_id UUID NOT NULL REFERENCES elevenlabs_voice_model(id)`
    - `created_at/updated_at` + trigger
  - **Seed:** thêm vài voice active vào `db/seed_dev.sql` (gồm voice mặc định `pNInz6obpgDQGcFmaJgB` để khớp `ELEVENLABS_VOICE_ID` hiện tại — đảm bảo TTS không vỡ khi user chưa chọn).

> Lựa chọn 1-1 (`user_id` PK) đơn giản, đủ cho AC-11-02; PUT = UPSERT (`ON CONFLICT (user_id) DO UPDATE`).

### Backend (speak-buddi-be/)
Tái dùng cơ chế Paid User đã có (KHÔNG tạo lại): guard `auth/deps.py::require_paid` (đọc claim `is_paid` trong JWT, đã set từ `user_repo.get_is_paid()` ở luồng login/refresh/S8.2). Pattern repo = raw SQL qua `sqlalchemy.text()` (giống `subscription_repo.py`/`payment_repo.py`).

- **`speak-buddi-be/schemas/voice.py`** (mới): `VoiceModelOut { id, voice_id, model_id, display_name, accent, gender, is_pro }`, `VoicePreferenceOut { voice_model_id, voice }` (voice = VoiceModelOut hoặc null), `SetVoicePreferenceRequest { voice_model_id: UUID }`.
- **`speak-buddi-be/repositories/voice_repo.py`** (mới):
  - `list_active_voice_models(db)` → các voice `is_active=true` order `sort_order`.
  - `get_voice_model(db, voice_model_id)` → 1 voice active (validate tồn tại + active khi PUT).
  - `get_user_preference(db, user_id)` → JOIN `user_voice_preference` + `elevenlabs_voice_model` (chỉ trả khi voice còn active; nếu voice bị soft-disable → coi như chưa chọn).
  - `set_user_preference(db, user_id, voice_model_id)` → UPSERT `ON CONFLICT (user_id)`.
  - `get_voice_id_for_user(db, user_id)` → trả `(voice_id, model_id)` của preference active, hoặc `None` (caller fallback default) — dùng cho TTS wiring.
- **`speak-buddi-be/routers/voice.py`** (mới), prefix `/api/voice`:
  - `GET /api/voice/models` — `Depends(require_paid)` → list voice active (AC-11-01). Free user → 403 (BR06/AC-11-03).
  - `GET /api/voice/preference` — `Depends(require_paid)` → preference hiện tại (hoặc null) để FE highlight thẻ đang chọn.
  - `PUT /api/voice/preference` — `Depends(require_paid)`, body `SetVoicePreferenceRequest` → validate voice tồn tại+active (404 nếu không) → UPSERT → trả `VoicePreferenceOut` (AC-11-02). Audit log `VOICE_PREF_SET user=%s voice_model=%s` (§4.5, không log gì khác).
- **`speak-buddi-be/main.py`** (sửa): import + `app.include_router(voice.router)`.
- **TTS wiring — `speak-buddi-be/services/tts_service.py`** (sửa): thêm tham số tùy chọn `voice_id`/`model_id` (giữ default = `ELEVENLABS_VOICE_ID` / `eleven_multilingual_v2` để không phá `/tts`,`/speak` hiện tại). `speak-buddi-be/routers/ai.py` (sửa): `/tts` & `/speak` lấy `current_user` (nếu có JWT) → `voice_repo.get_voice_id_for_user()` → truyền vào `text_to_audio_bytes()`; không có preference/không đăng nhập → default. **Lưu ý:** `/tts`,`/speak` hiện chưa bắt buộc JWT — giữ backward-compatible (preference chỉ áp dụng khi xác thực được user; không chặn để tránh phá luồng learning/speaking hiện có).

> **Quan trọng về 403 (AC-11-03):** `require_paid` đọc `is_paid` từ JWT. Sau khi free user vừa thanh toán xong (S8.2), JWT cũ vẫn `is_paid=false` đến khi refresh/login lại (đã ghi nhận ở S8.2). Không phải lỗi của S8.4 — màn voice cứ tin JWT; nếu cần realtime, FE gọi `/api/auth/refresh` sau upgrade (đã có sẵn). Ghi ở mục 6.
