# Plan — S9.5: Admin chỉnh/disable nội dung đã crawl + giữ lịch sử crawl

- **Epic:** EPIC-9 — Admin: Nội dung & Crawler Langeek (UC13 · F15, F17, F18 · BR07, BR10–BR12 · FR-17, FR-19)
- **Ưu tiên:** P2 (hoàn thiện vòng đời nội dung crawl sau auto-publish)
- **AC nguồn:** AC-13-06, BR11, FR-17, FR-19 (phần "Admin edit/disable after publish")
- **Ngày lập:** 2026-06-07

## 1. Mục tiêu & phạm vi

Hoàn thiện **quản trị nội dung Langeek sau khi auto-publish** (AC-13-06, FR-19): Admin có thể **chỉnh sửa** hoặc **vô hiệu hóa** topic/từ đã crawl; thay đổi **cấu hình crawler** (`content_source`) vẫn áp dụng cho các lần sync tương lai; **lịch sử crawl** (`content_crawl_job`, `content_crawl_log`) **không bị xóa** khi Admin disable/edit nội dung. Nội dung bị disable **không xuất hiện** ở roadmap / học từ / quiz học viên (BR11).

"Done" = Admin nhận diện nội dung `source=langeek` trên màn Topics/Vocabulary; chỉnh/sửa qua form CRUD hiện có; disable qua flow S9.2 **và** crawler sync tuần sau **không tự bật lại** bản ghi Admin đã disable; Admin có thể **kích hoạt lại** nội dung đã disable (bổ sung từ S9.2); chỉnh sửa Admin trên nội dung crawl **không bị ghi đè** bởi upsert crawler; thay đổi `is_enabled`/schedule trên `/admin/crawler` ảnh hưởng hành vi crawl tương lai; job/log crawl cũ vẫn xem được trên Admin Crawler.

> **Phạm vi loại trừ:**
> - **CRUD cơ bản + disable một chiều** → đã xong **S9.1/S9.2** (tái dùng UI/API, mở rộng logic).
> - **Happy path crawl + fallback lỗi** → **S9.3/S9.4** (verify-only, không làm lại).
> - **Hard delete** nội dung / xóa crawl history — không có trong SRS §5.3.
> - **Import CSV thay crawler** — TBD SRS; không làm trong story này.

### 3.1 Vấn đề cốt lõi cần giải quyết

| Hành vi hiện tại | Rủi ro | Cách xử lý S9.5 |
|---|---|---|
| `upsert_langeek_*` set `is_active=TRUE` mỗi lần crawl | Admin disable → crawl tuần sau **bật lại** | Thêm cờ **`admin_locked`**; crawler **không** đổi `is_active` nếu locked |
| `upsert_langeek_*` UPDATE toàn bộ field | Admin sửa `meaning_vi` → crawl **ghi đè** | Set `admin_locked=TRUE` khi Admin PUT; crawler **skip UPDATE** nội dung locked (chỉ INSERT mới) |
| S9.2 chỉ disable, không re-enable | Admin không khôi phục nội dung | `PATCH .../enable` hoặc `PUT` với `is_active=true` + clear lock tùy policy |
| FE không hiện `source=langeek` | Admin khó quản lý nội dung crawl | Badge + filter Source trên Topics/Vocabulary |

### Backend (speak-buddi-be/)

#### Schema — migration mới `db/schema_learning_s9_5.sql` (idempotent ALTER)

| Bảng | Cột mới | Mục đích |
|---|---|---|
| `topic` | `admin_locked BOOLEAN NOT NULL DEFAULT FALSE` | Admin đã disable hoặc chỉnh → crawler không ghi đè |
| `topic_word` | `admin_locked BOOLEAN NOT NULL DEFAULT FALSE` | Tương tự |

> Không thêm bảng mới. Crawl history giữ nguyên schema S9.3/S9.4.

#### Repository — `repositories/content_repo.py`

1. **`soft_delete_topic` / `soft_delete_word`** — bổ sung `admin_locked = TRUE` khi disable (phân biệt disable do crawler batch vs Admin).
2. **`update_topic` / `update_word`** — nếu `source='langeek'`, set `admin_locked = TRUE` sau UPDATE thành công.
3. **`enable_topic` / `enable_word`** (mới) — `SET is_active=TRUE, admin_locked=FALSE` (hoặc giữ lock nếu PO muốn "unlock riêng" — mặc định re-enable = unlock).
4. **`upsert_langeek_topic`** — nếu existing và `admin_locked=TRUE`: **skip UPDATE** (return existing, log warn trong crawl job). Nếu không locked: UPDATE như cũ **nhưng không force `is_active=TRUE` khi row đang inactive do admin** — thực tế locked row sẽ skip hết.
5. **`upsert_langeek_word`** — logic tương tự.
6. **`soft_disable_langeek_words_not_in_batch`** — **bỏ qua** word có `admin_locked=TRUE` (Admin disable thủ công không bị crawler "un-disable" ngược; word locked cũng không bị batch disable nếu Admin đã khóa — document trong implement log).

List/get queries: SELECT thêm `admin_locked` (optional trong response Admin).

#### Router — `routers/admin_content.py`

| Method | Path | Mô tả |
|---|---|---|
| PATCH | `/api/admin/topics/{id}/enable` | Re-enable topic (`is_active=true`, clear lock) — 404 nếu không tồn tại |
| PATCH | `/api/admin/words/{id}/enable` | Re-enable word |
| PATCH | `/api/admin/tests/{id}/enable` | Re-enable test (tests không có `source` crawl — tái dùng pattern S9.2, không cần `admin_locked`) |

Giữ nguyên DELETE disable (S9.2). Log: `ADMIN_ENABLE_*`, `ADMIN_LOCK_CRAWL_*`.

**Crawler config (AC-13-06 phần configuration):** không thêm endpoint — verify `PUT /api/admin/crawler/config` đã persist `is_enabled`, `schedule_cron`, retry policy; khi `is_enabled=false`, `check_crawl_permitted` chặn sync mới. Ghi test case trong verify.

#### Schemas — `schemas/learning.py`

- `TopicOut` / `TopicListItemOut` / `TopicWordOut`: thêm `admin_locked: bool` (optional default false cho backward compat).
- Không expose lock cho learner APIs.

#### Services — `services/crawler_publish.py`

- Sau upsert skip locked: `append_log` severity `info`/`warn` — "Bỏ qua topic/word {id} — admin_locked".
- Stats mới: `topics_skipped_locked`, `words_skipped_locked`.
